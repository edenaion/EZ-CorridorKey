Eine vollständige Desktop-GUI für [Niko Pueringers CorridorKey](https://github.com/nikopueringer/CorridorKey), den KI-Chroma-Keyer von Corridor Digital, der Vordergrund und Hintergrund physikalisch entmischt und dabei Haare, Bewegungsunschärfe und Transparenzen erhält.

Diese GUI ersetzt den CLI-Drag-and-drop-Workflow durch eine vollständige Desktop-Anwendung und behält dabei 100 % Abwärtskompatibilität (`python main.py --cli` startet weiterhin den ursprünglichen Assistenten).

## Installation

### Desktop-App-Installer (empfohlen)

**Keine Lust auf Python, git oder Kommandozeile?** Ein vollständiger Windows-Installer, eine portable exe und ein macOS-`.pkg` sind verfügbar. Alles ist optional und kostenlos. Spenden unterstützen die aktive Entwicklung.

[![Auf Gumroad herunterladen](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

Der Installer enthält alles: Python-Runtime, KI-Modelle, GPU-Bibliotheken. Kein Setup nötig. Installieren und starten.

### Terminal-Installation (CLI) (Windows / macOS / Linux)

1. Klonen oder laden Sie dieses Repository herunter.
2. Der Ein-Klick-Pfad stellt verwaltetes Python 3.11 automatisch bereit und verwendet es. Sie müssen Python also nicht vorab installieren, nur um `1-install` zu nutzen.
3. Führen Sie den Installer für Ihre Plattform aus:
   ☼ **Windows:** Doppelklick auf `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. Der Installer übernimmt alles: verwaltetes Python, virtuelle Umgebung, Abhängigkeiten (inklusive passendem PyTorch-Backend für Ihre GPU, falls verfügbar), Prüfung und Modelldownloads.
5. Zum Starten: Doppelklick auf `2-start.bat` (Windows) oder `./2-start.sh` (macOS/Linux).

**Voraussetzungen:**

☼ Für den Ein-Klick-Installer: Kein vorinstalliertes Python erforderlich

☼ Für manuelle Installationen: [Python 3.10–3.13](https://python.org) (3.14 wird noch nicht unterstützt)

☼ **Windows/Linux:** NVIDIA-GPU mit CUDA-Unterstützung (8 GB+ VRAM empfohlen). Halten Sie Ihren Treiber aktuell. Der Installer prüft die torch-Runtime und stoppt mit Diagnosen, statt Sie still auf dem falschen Backend zu lassen.

☼ **macOS:** Apple Silicon (M1+). CorridorKey-Inferenz läuft nativ über MLX (1,5 bis 2x schneller als MPS). GPU-intensive Alpha-Generatoren (SAM2, GVM, VideoMaMa, MatAnyone2) laufen auf MPS, sind aber deutlich langsamer. Auf dem Mac wird der Import fertiger Alpha-Matten empfohlen.

**Was der Installer macht:**

☼ Prüft auf [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (C++-Compiler, von OpenEXR benötigt) und bietet eine automatische Installation an, falls sie fehlen

☼ Installiert [uv](https://docs.astral.sh/uv/) und stellt verwaltetes Python 3.11 für den Installer-Pfad bereit

☼ Erstellt eine virtuelle Umgebung `.venv` im Projektordner

☼ Installiert das passende PyTorch-Backend für Ihre Plattform/GPU und prüft die resultierende torch-Runtime, bevor Erfolg gemeldet wird

☼ Lädt [FFmpeg](https://ffmpeg.org/) lokal herunter und installiert es, falls es nicht im PATH gefunden wird (für Videoimport)

☼ Lädt den CorridorKey-Modell-Checkpoint herunter (383 MB, erforderlich)

☼ Installiert optional SAM2-Tracking-Unterstützung und lädt den Standard-Base+-Checkpoint vorab herunter (324 MB)

☼ Lädt optional die Alpha-Hint-Generatoren BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB) und VideoMaMa (~37 GB) herunter

☼ Erstellt eine Desktop-Verknüpfung (optional)

**Aktualisierung:**

☼ **Benutzer des Windows Desktop App Installers:** Die App prüft automatisch auf Updates. Wenn eine neue Version verfügbar ist, klicken Sie in der App auf den Update-Button. Sie lädt einen schlanken Patch herunter und startet neu.

☼ **Benutzer der macOS Desktop App:** Die App prüft automatisch auf Updates. Wenn eine neue Version verfügbar ist, klicken Sie in der App auf den Update-Button. Sie lädt einen schlanken Patch herunter und startet neu.

☼ **CLI-Benutzer:** Doppelklick auf `3-update.bat` (Windows) oder `./3-update.sh` ausführen (macOS/Linux). Dadurch wird der aktuelle Code per git geholt oder ein ZIP geladen, falls git nicht verfügbar ist.

> **Hinweis:** Das Update-ZIP auf GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) ist nur für Benutzer des Windows Desktop App Installers gedacht. Es patcht eine bestehende Installation. CLI-Benutzer sollten weiter `3-update.bat` / `3-update.sh` verwenden.

### Alternative Installation: Docker

Für Linux-Benutzer oder Remote-/Cloud-Setups kann EZ-CorridorKey in Docker mit browserbasiertem Zugriff über noVNC laufen. Siehe [docker/README.md](docker/README.md) für Setup-Anweisungen. Für Windows und macOS wird die native Installation oben empfohlen.

## Deinstallation

### Desktop-App-Installer

**Windows:**
☼ Öffnen Sie Einstellungen > Apps > Installierte Apps

☼ Suchen Sie **EZ-CorridorKey** und klicken Sie auf Uninstall

☼ Dadurch werden die Anwendung und die Startmenü-Verknüpfung entfernt

☼ Ihre Projekte und heruntergeladenen Modelle liegen in `%APPDATA%\EZ-CorridorKey\`. Löschen Sie diesen Ordner, um alle Benutzerdaten zu entfernen.

**macOS:**
☼ Ziehen Sie `/Applications/EZ-CorridorKey.app` in den Papierkorb

☼ Ihre Projekte, Einstellungen und heruntergeladenen Modelle liegen in `~/Library/Application Support/EZ-CorridorKey/`. Löschen Sie diesen Ordner, um alle Benutzerdaten zu entfernen.

### CLI-Installation (git clone)

☼ Löschen Sie den geklonten Repository-Ordner (z. B. `EZ-CorridorKey/`). Das umfasst die virtuelle Umgebung `.venv`, heruntergeladene Modelle und alle Projektdaten in `Projects/`.

☼ Wenn Sie bei der Installation eine Desktop-Verknüpfung erstellt haben, löschen Sie sie manuell.

☼ Die CLI-Installation ändert keine systemweiten Dateien. Es gibt nichts Weiteres aufzuräumen.

### Hugging-Face-Modellcache

Einige optionale Modelle (BiRefNet, SAM2) werden über Hugging Face Hub heruntergeladen und außerhalb des Projektordners zwischengespeichert. Um diesen Speicherplatz freizugeben:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Dieser Cache wird von allen Anwendungen geteilt, die Hugging Face nutzen. Wenn Sie andere KI-Tools verwenden, löschen Sie nur die jeweiligen Modellordner (z. B. `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`) statt des gesamten `hub/`-Verzeichnisses.

## Schnellstart

### 1. Importieren

Legen Sie eine Videodatei auf dem Willkommensbildschirm ab (oder File > Import Clips > Import Video). Das Video wird automatisch in eine hochwertige Bildsequenz extrahiert:

1. **FFmpeg extrahiert** jeden Frame als **EXR Half-Float** (ZIP16-Komprimierung)
2. **Rekomprimierungsdurchgang** konvertiert ZIP16 → **DWAB** (verlustbehaftete VFX-Standardkomprimierung, ~4× kleiner)

Dieser Zwei-Pass-Ansatz erhält die volle Fließkommapräzision des Videodecoders: keine 8-Bit-Quantisierung, kein Banding. Selbst 8-Bit-Quellvideo profitiert davon, weil FFmpegs interne YUV→RGB-Konvertierung in float bleibt und Rundungsfehler vermeidet, die sich in Integer-Pipelines aufsummieren. Die DWAB-komprimierten EXR-Frames sind meist ähnlich groß wie PNG, behalten aber den 16-Bit-Half-Float-Dynamikumfang.

Die Rekompression läuft in einem separaten Prozess, damit die Oberfläche während der Extraktion vollständig reaktionsfähig bleibt.

### 2. Alpha-Hint generieren

Ihr Clip startet im **RAW**-Status (graues Badge). Vor der Inferenz benötigen Sie einen Alpha-Hint.

**Option A - Chroma Key (manuell, kein KI-Modell):**
Klicken Sie im Parameterpanel auf **CHROMA KEY**. Farbdifferenz-Keyer für Greenscreen- oder Bluescreen-Material. Drücken Sie **E**, um die Pipette zu aktivieren, und ziehen Sie dann über den Screen, um einen Farbbereich aufzunehmen. Passen Sie Key Strength, Clip Black und Clip White an, um die Matte zu verfeinern. Klicken Sie auf **GENERATE**, um den Alpha-Hint zu erzeugen.

**Option B - Ein-Klick-Alpha-Generatoren:**
☼ **GVM Auto** - klicken Sie im Parameterpanel auf **GVM AUTO**. Funktioniert sehr gut für die meisten Greenscreen-Aufnahmen mit Personen.

☼ **BiRefNet** (empfohlen) - klicken Sie im Parameterpanel auf **BIREFNET**. Schnell, präzise und gut für viele Arten von Material geeignet.

**Option C - MatAnyone2 / VideoMaMa:**
Diese Modelle benötigen einen Masken-Hint. Es gibt zwei Wege:

*C1. Malen + Tracken (von Grund auf):*
1. Drücken Sie **1**, um den Vordergrundmodus zu aktivieren (grün)
2. Malen Sie auf einigen Keyframes über das Motiv
3. Drücken Sie **2**, um in den Hintergrundmodus zu wechseln (rot)
4. Malen Sie über Hintergrundbereiche
5. Klicken Sie auf **TRACK MASK**, um einen dichten SAM2-Maskentrack zu erzeugen
6. Klicken Sie im Parameterpanel auf **MATANYONE2** oder **VIDEOMAMA**

*C2. Maske importieren (eigene Maske):*
1. Klicken Sie auf den **+**-Button neben **VIDEOMAMA**, um eine vorbereitete Maskensequenz oder ein Video zu importieren
2. Klicken Sie auf **MATANYONE2** oder **VIDEOMAMA**, um das Modell auszuführen

**Option D - Alpha importieren (eigenes Alpha):**
Wenn Sie bereits Alpha-Matten aus einem anderen Tool haben (Rotobrush, Silhouette, Resolve, Nuke usw.), klicken Sie im Parameterpanel auf **IMPORT ALPHA** und wählen Sie entweder einen Bildordner oder eine Matte-Videodatei.

☼ Unterstützte Bildformate: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Unterstützter Alpha-Video-Pfad: Standardvideodateien, die vom normalen Clip-Importer akzeptiert werden (zum Beispiel **MOV** oder **MP4**)

☼ Bilder und sichtbare Matte-Videos sollten **Graustufen** sein (weiß = Vordergrund, schwarz = Hintergrund)

☼ Die Frameanzahl sollte zu Ihrer Eingabesequenz passen

☼ Nicht-PNG-Standbilder werden beim Import automatisch in Graustufen-PNG konvertiert

☼ Importierte Alpha-Videos werden in Graustufen-Frames `AlphaHint/*.png` decodiert, damit sie denselben Downstream-Pfad wie Bildsequenz-Hints nutzen

☼ Importierte Dateien werden in den Ordner `AlphaHint/` des Clips kopiert und der Clip wechselt in den **READY**-Status

Sie können jederzeit erneut importieren. Wenn der Clip bereits Alpha-Hints hat, werden Sie gefragt, ob sie überschrieben werden sollen.

### 3. Inferenz starten

Ihr Clip ist jetzt **READY** (gelbes Badge). Passen Sie die Parameter nach Bedarf an und klicken Sie dann auf **RUN INFERENCE** (oder **Ctrl+R**).

### Batch-Pipeline (unbeaufsichtigt)

Wählen Sie mehrere Clips im I/O-Tray aus (Ctrl+Klick oder Shift+Klick) und klicken Sie dann in der Statusleiste auf **RUN PIPELINE**. Das System:

1. Erkennt den Status und die Malstriche jedes Clips
2. Führt **TRACK MASK** und danach **VideoMaMa** für bemalte Clips aus
3. Führt **GVM Auto** für unbemalte RAW-Clips aus
4. Verkettet die **Inferenz**, sobald jede Alpha-Generierung abgeschlossen ist

Die gesamte Pipeline kann abgebrochen werden (Esc) und ist checkpointfähig. Bei Unterbrechung setzt ein Neustart dort fort, wo jeder Clip aufgehört hat.

### 4. Prüfen

Wechseln Sie zwischen den Anzeigemodi, um Ergebnisse zu prüfen:

☼ **COMP** - Key über Schachbrettmuster

☼ **FG** - auf grünes Fringing prüfen

☼ **MATTE** - Alpha-Qualität prüfen

☼ **PROCESSED** - Produktions-RGBA

Ausgaben werden während der Inferenz in die `Output/`-Unterverzeichnisse des Projekts geschrieben (konfigurierbar, siehe [Benutzerdefiniertes Ausgabeverzeichnis](#benutzerdefiniertes-ausgabeverzeichnis)).

**Verhalten von Color Space**

☼ Der **linke INPUT-Viewer** zeigt immer CorridorKeys aktuelle Interpretation der Quelle. Wenn die Eingabe dort falsch aussieht, basieren auch Ihre zukünftigen Inferenzergebnisse und Exporte auf dieser falschen Interpretation.

☼ Das Ändern von **Color Space** vor dem Klick auf **RUN INFERENCE** beeinflusst, wie die nächste Live-Vorschau und der nächste Exportlauf erzeugt werden.

☼ Das Ändern von **Color Space**, nachdem Ausgaben bereits existieren, schreibt diese Dateien auf der Festplatte **nicht** neu. Es aktualisiert nur den Viewer und die Live-Vorschau. Um diese neue Interpretation in gespeicherten Dateien zu behalten, führen Sie die Inferenz erneut aus.

☼ CorridorKey erkennt den Farbraum nach Möglichkeit automatisch aus Dateityp und Metadaten. Sie können ihn überschreiben, wenn der INPUT-Viewer nicht repräsentativ aussieht.

☼ Wenn **Live Preview** aktiviert ist, kann die erste Anpassung nach einem frischen Start kurz pausieren, während die Inferenz-Engine geladen wird.

## Frame-Scrubber

Der Scrubber unter dem Doppelviewer bietet:

☼ **Transport-Buttons:** Erster Frame, Schritt zurück, Wiedergabe/Pause, Schritt vor, letzter Frame

☼ **Wiedergabe ist CAPPED:** Drücken der Leertaste spielt Material mit einer hart codierten Rate von 3FPS ab. Das ist absichtlich so, die Dateien sind groß.

☼ **Abdeckungsleiste:** Drei farbcodierte Spuren zeigen, welche Frames Malstriche (grün), Alpha-Hints (weiß) und Inferenzausgabe (gelb) haben

☼ **In/Out-Marker:** Drücken Sie **I** / **O**, um einen Teilbereich für die Verarbeitung festzulegen. Wenn gesetzt, ändert sich der RUN-Button zu "RUN SELECTED" und die Wiedergabe läuft innerhalb des Bereichs in Schleife.

## Hardwareanforderungen

### Windows / Linux (NVIDIA CUDA)

|          | Minimum       | Empfohlen          | Komfortabel                   |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 GB          | 12 GB              | 16 GB+                        |

#### VRAM-Modi

| Modus                 | VRAM-Nutzung | 4K-Geschwindigkeit | Funktionsweise                           |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Full-frame-Refiner, vollständiges torch.compile |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | 512×512 gekachelter Refiner, selektive Kompilierung |

Der Modus wird anhand des verfügbaren VRAM automatisch erkannt. Überschreiben mit `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

> Gilt für v2.0.0 und Quellinstallationen. Patch-Releases wie 2.1.2 werden als automatisches In-App-Update ausgeliefert, nicht als neuer Mac-Installer.

|             | Minimum       | Empfohlen                                        |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (automatisch erkannt, falls corridorkey-mlx installiert ist) |

CorridorKey-Inferenz wählt automatisch das schnellste verfügbare Backend: MLX (1,5 bis 2x schneller), wenn `corridorkey-mlx` und ein `.safetensors`-Checkpoint vorhanden sind, sonst PyTorch MPS. Überschreiben mit `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Alpha-Generatoren (SAM2, GVM, VideoMaMa, MatAnyone2) laufen immer auf PyTorch MPS. Für diese Modelle gibt es keine MLX-Ports. Für die beste Mac-Erfahrung importieren Sie fertige Alpha-Matten aus After Effects, DaVinci Resolve oder Nuke.

### Signierte Updates

Ab v1.10.0 enthält jede GitHub-Release ein signiertes Manifest (`manifest.json` + `manifest.json.sig`). Wenn der In-App-Updater eine neue Version herunterlädt, prüft er die Ed25519-Signatur und den SHA-256-Hash, bevor das Update angewendet wird. Wenn eine Prüfung fehlschlägt, wird das Update abgelehnt und der Benutzer gewarnt.

Das bedeutet: Selbst wenn das GitHub-Konto kompromittiert wäre, könnte ein Angreifer installierten Benutzern kein bösartiges Update ausliefern, ohne auch den Offline-Signaturschlüssel zu besitzen.

Der Prüfcode ist Open Source: [`backend/update_verify.py`](backend/update_verify.py).

### Prüfsummen

Jede Release enthält eine Datei `SHA256SUMS.txt` mit dem SHA-256-Hash jedes Release-Artefakts. Laden Sie sie in denselben Ordner wie die heruntergeladene Datei und folgen Sie den Schritten für Ihr System.

**Windows (PowerShell):**

```powershell
# Im Download-Ordner (Umschalt + Rechtsklick > "PowerShell-Fenster hier öffnen")
Get-FileHash .\EZ-CorridorKey-2.1.0-Windows-x64-Setup.exe -Algorithm SHA256
# Öffnen Sie SHA256SUMS.txt, suchen Sie die Zeile mit diesem Dateinamen und vergleichen Sie den Hash.
# Die beiden Zeichenfolgen müssen exakt übereinstimmen. Falls sie abweichen, führen Sie die Datei nicht aus, sondern laden Sie sie erneut herunter.
```

**macOS (Terminal):**

```bash
cd ~/Downloads   # wo Sie die Dateien gespeichert haben
shasum -a 256 -c SHA256SUMS.txt --ignore-missing
# Erwartet: EZ-CorridorKey-2.0.0-macOS-arm64.dmg: OK
# Falls FAILED angezeigt wird, ist die Datei beschädigt oder manipuliert. Laden Sie sie erneut herunter.
```

**Linux:**

```bash
cd ~/Downloads
sha256sum -c SHA256SUMS.txt --ignore-missing
# Neben jeder heruntergeladenen Datei sollte OK stehen.
```

## Lokalisierung

EZ-CorridorKey unterstützt Übersetzungen in jede Sprache. Alle UI-Strings werden in standardmäßige Qt-`.ts`-Dateien extrahiert, die Übersetzer mit [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (kostenlos) oder einem beliebigen Texteditor bearbeiten können.

**So fügen Sie Ihre Sprache hinzu:**

1. Kopieren Sie `ui/translations/corridorkey_en.ts` zu Ihrem Sprachcode (z. B. `corridorkey_fr.ts` für Französisch)
2. Öffnen Sie die neue Datei in Qt Linguist und tragen Sie die Übersetzungen ein
3. Kompilieren Sie sie: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Reichen Sie einen Pull Request mit den Dateien `.ts` und `.qm` ein

Vollständige Anweisungen mit Beispielen stehen in [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Aktuelle Sprachen:** Englisch (eingebaut). Sie möchten Ihre Sprache hier sehen? PRs sind willkommen.
