## Установка

### Установщик настольного приложения (рекомендуется)

**Не хотите разбираться с Python, git или командной строкой?** Доступны полный установщик Windows, портативный exe и macOS `.pkg`. Это полностью необязательно и бесплатно. Пожертвования помогают поддерживать активную разработку.

[![Скачать на Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

Установщик включает все необходимое: runtime Python, AI-модели, GPU-библиотеки. Дополнительная настройка не требуется. Просто установите и запустите.

### Установка через терминал (CLI) (Windows / macOS / Linux)

1. Склонируйте или скачайте этот репозиторий.
2. Однокликовый путь автоматически подготавливает и использует управляемый Python 3.11, поэтому вам не нужно заранее устанавливать Python только ради `1-install`.
3. Запустите установщик для вашей платформы:
   ☼ **Windows:** дважды щелкните `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. Установщик делает все: управляемый Python, виртуальная среда, зависимости (включая правильный backend PyTorch для вашего GPU, если он доступен), проверка и загрузка моделей.
5. Для запуска дважды щелкните `2-start.bat` (Windows) или выполните `./2-start.sh` (macOS/Linux).

**Требования:**

☼ Для однокликового установщика: заранее установленный Python не нужен

☼ Для ручной установки: [Python 3.10–3.13](https://python.org) (3.14 пока не поддерживается)

☼ **Windows/Linux:** GPU NVIDIA с поддержкой CUDA (рекомендуется 8 GB+ VRAM). Держите драйвер свежим. Установщик проверяет runtime torch и остановится с диагностикой, если backend выбран неверно.

☼ **macOS:** Apple Silicon (M1+). Инференс CorridorKey нативно работает через MLX (в 1.5–2 раза быстрее MPS). GPU-интенсивные генераторы альфы (SAM2, GVM, VideoMaMa, MatAnyone2) работают на MPS, но заметно медленнее. На Mac рекомендуется импортировать заранее подготовленные альфа-маски.

**Что делает установщик:**

☼ Проверяет [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (компилятор C++ нужен для OpenEXR), при отсутствии предлагает автоустановку

☼ Устанавливает [uv](https://docs.astral.sh/uv/) и подготавливает управляемый Python 3.11 для пути установки

☼ Создает виртуальную среду `.venv` в папке проекта

☼ Устанавливает правильный backend PyTorch для вашей платформы/GPU и проверяет полученный runtime torch перед сообщением об успехе

☼ Если [FFmpeg](https://ffmpeg.org/) не найден в PATH, скачивает и устанавливает его локально (используется для импорта видео)

☼ Скачивает checkpoint модели CorridorKey (383 MB, обязателен)

☼ При необходимости устанавливает поддержку трекинга SAM2 и заранее скачивает стандартный checkpoint Base+ (324 MB)

☼ При необходимости скачивает генераторы альфа-подсказок BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB) и VideoMaMa (~37 GB)

☼ Создает ярлык на рабочем столе (необязательно)

**Обновление:**

☼ **Пользователи установщика настольного приложения Windows:** приложение автоматически проверяет обновления. Когда доступна новая версия, нажмите кнопку обновления в приложении. Оно скачает легкий патч и перезапустится.

☼ **Пользователи настольного приложения macOS:** Приложение автоматически проверяет обновления. Когда доступна новая версия, нажмите кнопку обновления в приложении. Оно скачает легкий патч и перезапустится.

☼ **Пользователи CLI:** дважды щелкните `3-update.bat` (Windows) или выполните `./3-update.sh` (macOS/Linux). Это подтянет последний код через git или скачает ZIP, если git недоступен.

> **Примечание:** ZIP обновления в GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) предназначен только для пользователей установщика настольного приложения Windows. Он патчит существующую установку. Пользователям CLI следует продолжать использовать `3-update.bat` / `3-update.sh`.

### Альтернативная установка: Docker

Пользователи Linux и удаленные/облачные среды могут запускать EZ-CorridorKey внутри Docker с доступом из браузера через noVNC. Инструкции по настройке см. в [docker/README.md](docker/README.md). Для Windows и macOS рекомендуется нативная установка выше.

## Удаление

### Установщик настольного приложения

**Windows:**
☼ Откройте Settings > Apps > Installed Apps

☼ Найдите **EZ-CorridorKey** и нажмите Uninstall

☼ Это удалит приложение и ярлык в меню Start

☼ Ваши проекты и скачанные модели хранятся в `%APPDATA%\EZ-CorridorKey\`. Удалите эту папку, чтобы убрать все пользовательские данные.

**macOS:**
☼ Перетащите `/Applications/EZ-CorridorKey.app` в Trash

☼ Ваши проекты, настройки и скачанные модели хранятся в `~/Library/Application Support/EZ-CorridorKey/`. Удалите эту папку, чтобы убрать все пользовательские данные.

### Установка CLI (git clone)

☼ Удалите папку склонированного репозитория (например, `EZ-CorridorKey/`). Она включает виртуальную среду `.venv`, скачанные модели и все проектные данные внутри `Projects/`.

☼ Если во время установки вы создали ярлык на рабочем столе, удалите его вручную.

☼ Установка CLI не изменяет системные файлы. Больше ничего чистить не нужно.

### Кэш моделей Hugging Face

Некоторые необязательные модели (BiRefNet, SAM2) скачиваются через Hugging Face Hub и кэшируются вне папки проекта. Чтобы вернуть это место на диске:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Этот кэш используется всеми приложениями, которые работают с Hugging Face. Если вы используете другие AI-инструменты, удаляйте только конкретные папки моделей (например, `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`), а не весь каталог `hub/`.

## Быстрый старт

### 1. Импорт

Перетащите видеофайл на экран приветствия (или File > Import Clips > Import Video). Видео автоматически извлекается в высококачественную последовательность изображений:

1. **FFmpeg извлекает** каждый кадр как **EXR half-float** (сжатие ZIP16)
2. **Проход повторного сжатия** преобразует ZIP16 → **DWAB** (стандартное для VFX сжатие с потерями, примерно в 4 раза меньше)

Этот двухпроходный подход сохраняет полную точность floating-point из видеодекодера. Нет 8-битного квантования и полос. Даже 8-битное исходное видео получает пользу, потому что внутренняя конвертация YUV→RGB в FFmpeg остается в float и избегает ошибок округления, которые накапливаются в integer-пайплайнах. EXR-кадры с DWAB-сжатием обычно сопоставимы по размеру с PNG и сохраняют 16-битный half-float динамический диапазон.

Повторное сжатие выполняется в отдельном процессе, поэтому UI остается полностью отзывчивым во время извлечения.

### 2. Генерация альфа-подсказки

Клип начинается в состоянии **RAW** (серый бейдж). Перед запуском инференса нужна альфа-подсказка.

**Вариант A: хромакей (вручную, без AI-модели):**
Нажмите **CHROMA KEY** на панели параметров. Это keyer по разнице цвета для материала с зеленым или синим экраном. Нажмите **E**, чтобы активировать пипетку, затем протяните по экрану для выборки диапазона цветов. Настройте Key Strength, Clip Black и Clip White, чтобы уточнить маску. Нажмите **GENERATE**, чтобы создать альфа-подсказку.

**Вариант B: генераторы альфы в один клик:**
☼ **GVM Auto:** нажмите **GVM AUTO** на панели параметров. Хорошо работает для большинства материалов с зеленым экраном и людьми.

☼ **BiRefNet** (рекомендуется): нажмите **BIREFNET** на панели параметров. Быстро, точно и хорошо работает с широким набором материалов.


**Вариант C: MatAnyone2 / VideoMaMa:**
Этим моделям нужна маска-подсказка. Есть два способа ее дать:

*C1. Рисование + трекинг (с нуля):*
1. Нажмите **1**, чтобы включить режим переднего плана (зеленый)
2. Закрасьте объект на нескольких ключевых кадрах
3. Нажмите **2**, чтобы перейти в режим фона (красный)
4. Закрасьте области фона
5. Нажмите **TRACK MASK**, чтобы сгенерировать плотный трек маски SAM2
6. Нажмите **MATANYONE2** или **VIDEOMAMA** на панели параметров

*C2. Импорт маски (свой файл):*
1. Нажмите кнопку **+** рядом с **VIDEOMAMA**, чтобы импортировать готовую последовательность масок или видео
2. Нажмите **MATANYONE2** или **VIDEOMAMA**, чтобы запустить модель

**Вариант D: импорт альфы (свой файл):**
Если у вас уже есть альфа-маски из другого инструмента (Rotobrush, Silhouette, Resolve, Nuke и т. д.), нажмите **IMPORT ALPHA** на панели параметров и выберите папку изображений или видео с маской.

☼ Поддерживаемые форматы изображений: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Поддерживаемый путь alpha-video: стандартные видеофайлы, принимаемые обычным импортером клипов (например, **MOV** или **MP4**)

☼ Изображения и видимые видео с маской должны быть **grayscale** (белый = передний план, черный = фон)

☼ Число кадров должно совпадать с входной последовательностью

☼ Неподвижные изображения не в PNG автоматически конвертируются в grayscale PNG при импорте

☼ Импортированные alpha-видео декодируются в grayscale-кадры `AlphaHint/*.png`, чтобы использовать тот же дальнейший путь, что и подсказки из последовательности изображений

☼ Импортированные файлы копируются в папку клипа `AlphaHint/`, а клип переходит в состояние **READY**

Можно импортировать повторно в любой момент. Если у клипа уже есть альфа-подсказки, приложение спросит, перезаписать ли их.

### 3. Запуск инференса

Клип теперь **READY** (желтый бейдж). Настройте параметры при необходимости, затем нажмите **RUN INFERENCE** (или **Ctrl+R**).

### Пакетная обработка (без присмотра)

Выберите несколько клипов в I/O-лотке (Ctrl+щелчок или Shift+щелчок), затем нажмите **RUN PIPELINE** в строке состояния. Система автоматически:

1. Определяет состояние каждого клипа и штрихи рисования
2. Запускает **TRACK MASK**, а затем **VideoMaMa** для нарисованных клипов
3. Запускает **GVM Auto** для ненарисованных RAW-клипов
4. Запускает **инференс** после завершения каждой генерации альфы

Весь пайплайн можно отменить (Esc), он поддерживает контрольные точки. Если его прервать, перезапуск продолжит с места остановки каждого клипа.

### 4. Проверка

Переключайте режимы просмотра, чтобы проверить результат:

☼ **COMP:** key поверх шахматного фона

☼ **FG:** проверка зеленой окантовки

☼ **MATTE:** проверка качества альфы

☼ **PROCESSED:** производственный RGBA

Во время инференса вывод пишется в подкаталоги `Output/` проекта (настраивается, см. [Пользовательская папка вывода](#пользовательская-папка-вывода)).

**Поведение Color Space**

## Требования к оборудованию

### Windows / Linux (NVIDIA CUDA)

|          | Минимум       | Рекомендуется      | Комфортно                     |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 GB          | 12 GB              | 16 GB+                        |
| **GPU**  | NVIDIA (CUDA) | Ampere+ (RTX 30xx) | Ada/Blackwell (RTX 40xx/50xx) |

#### Режимы VRAM

| Режим                 | Использование VRAM | Скорость 4K | Как работает                             |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Полноразмерный рефайнер, полный torch.compile   |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | Рефайнер плитками 512×512, выборочная компиляция |

Режим автоматически определяется по доступной VRAM. Переопределение: `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

|             | Минимум       | Рекомендуется                                   |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (встроенный, автоопределение)                 |

> Применяется к v2.0.0 и исходным установкам. Патч-релизы, такие как 2.1.2, поставляются как встроенное автообновление внутри приложения, а не в виде нового установщика для Mac.

Инференс CorridorKey автоматически выбирает самый быстрый доступный backend: встроенный движок MLX (примерно в 2 раза быстрее MPS, работает в float16), если есть checkpoint `.mlx.safetensors`, иначе PyTorch MPS. Оба checkpoint — зеленый и синий MLX — скачиваются через мастер установки. Переопределение: `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Генераторы альфы (SAM2, GVM, VideoMaMa, MatAnyone2) всегда работают на PyTorch MPS. Для этих моделей нет портов MLX. Для лучшего опыта на Mac импортируйте заранее подготовленные альфа-маски из After Effects, DaVinci Resolve или Nuke.

### Подписанные обновления

Начиная с v1.10.0 каждый релиз GitHub поставляется с подписанным манифестом (`manifest.json` + `manifest.json.sig`). Когда встроенный updater скачивает новую версию, он проверяет подпись Ed25519 и хэш SHA-256 перед применением обновления. Если любая проверка не проходит, обновление отклоняется, а пользователь получает предупреждение.

Это значит, что даже при компрометации аккаунта GitHub атакующий не сможет отправить вредоносное обновление установленным пользователям без офлайн-ключа подписи.

Код проверки открыт: [`backend/update_verify.py`](backend/update_verify.py).

### Контрольные суммы

Каждый релиз включает файл `SHA256SUMS.txt` со списком SHA-256 хэшей всех артефактов релиза. Поместите `SHA256SUMS.txt` в ту же папку, что и скачанный файл, затем выполните следующие шаги для вашей системы.

**Windows (PowerShell):**

```powershell
# В папке загрузки (Shift + правый щелчок > "Open PowerShell window here")

# Откройте SHA256SUMS.txt, найдите строку с этим именем файла и сравните хэш.
# Две строки должны совпадать точно. Если они различаются, не запускайте файл, скачайте его снова.
```

**macOS (Terminal):**

```bash
cd ~/Downloads   # туда, где вы сохранили файлы
shasum -a 256 -c SHA256SUMS.txt --ignore-missing
# Ожидается: EZ-CorridorKey-2.0.0-macOS-arm64.dmg: OK
# Если написано FAILED, файл поврежден или подменен, скачайте его снова.
```

**Linux:**


## Локализация

EZ-CorridorKey поддерживает перевод на любой язык. Все строки UI извлекаются в стандартные Qt-файлы `.ts`, которые переводчики могут редактировать в [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (бесплатно) или любом текстовом редакторе.

**Как добавить ваш язык:**

1. Скопируйте `ui/translations/corridorkey_en.ts` в код вашего языка (например, `corridorkey_fr.ts` для французского)
2. Откройте новый файл в Qt Linguist и заполните переводы
3. Скомпилируйте его: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Отправьте pull request с файлами `.ts` и `.qm`

Полные инструкции с примерами находятся в [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Текущие языки:** английский плюс немецкий, испанский, французский, хинди, индонезийский, итальянский, японский, корейский, польский, португальский (Бразилия), русский, турецкий, украинский, вьетнамский, китайский (упрощенный) и китайский (традиционный, Тайвань). Выберите нужный в Edit > Preferences. Хотите видеть здесь свой язык или улучшить перевод? PR приветствуются.
