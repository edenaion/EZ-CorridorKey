**不想处理 Python、git 或命令行？** 提供完整 Windows 安装程序、便携 exe 和 macOS `.pkg`。完全可选且免费，捐赠可支持持续开发。

安装程序包含所有内容：Python 运行时、AI 模型、GPU 库，无需额外设置。安装后直接运行即可。

### 终端 (CLI) 安装 (Windows / macOS / Linux)

1. 克隆或下载此仓库。
2. 一键路径会自动准备并使用托管 Python 3.11，因此你无需为了使用 `1-install` 而预先安装 Python。
3. 运行适合你平台的安装程序：
   ☼ **Windows：** 双击 `1-install.bat`

   ☼ **macOS / Linux：** `chmod +x 1-install.sh && ./1-install.sh`

4. 安装程序会处理所有内容：托管 Python、虚拟环境、依赖项（可用时包含适合你 GPU 的正确 PyTorch 后端）、验证和模型下载。
5. 启动：双击 `2-start.bat`（Windows）或运行 `./2-start.sh`（macOS/Linux）。

**先决条件：**

☼ 一键安装程序：无需预装 Python

☼ 手动安装：[Python 3.10–3.13](https://python.org)（尚不支持 3.14）

☼ **Windows/Linux：** 支持 CUDA 的 NVIDIA GPU（建议 8 GB+ VRAM）。请保持驱动为最新版本。安装程序会验证 torch 运行时，如后端错误，会给出诊断并停止，而不是静默留下错误配置。

☼ **macOS：** Apple Silicon (M1+)。CorridorKey 推理通过 MLX 原生运行（比 MPS 快 1.5–2 倍）。GPU 密集型 Alpha 生成器（SAM2、GVM、VideoMaMa、MatAnyone2）在 MPS 上运行，但速度明显更慢。Mac 上建议导入预制 Alpha 遮罩。

**安装程序执行内容：**

☼ 检查 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)（OpenEXR 需要 C++ 编译器），缺失时提供自动安装

☼ 安装 [uv](https://docs.astral.sh/uv/)，并为安装路径准备托管 Python 3.11

☼ 在项目文件夹中创建 `.venv` 虚拟环境

☼ 为你的平台/GPU 安装正确的 PyTorch 后端，并在报告成功前验证得到的 torch 运行时

☼ 如果 PATH 中找不到 [FFmpeg](https://ffmpeg.org/)，则在本地下载并安装（用于视频导入）

☼ 下载 CorridorKey 模型 checkpoint（383 MB，必需）

☼ 可选安装 SAM2 跟踪支持，并预下载默认 Base+ checkpoint（324 MB）

☼ 可选下载 BiRefNet（约 940 MB）、MatAnyone2（约 135 MB）、GVM（约 6 GB）和 VideoMaMa（约 37 GB）Alpha 提示生成器

☼ 创建桌面快捷方式（可选）

**更新：**

☼ **Windows 桌面应用安装程序用户：** 应用会自动检查更新。新版本可用时，点击应用内更新按钮。它会下载轻量补丁并重新启动。

☼ **macOS 桌面应用用户：** 应用会自动检查更新。新版本可用时，点击应用内更新按钮。它会下载轻量补丁并重新启动。

☼ **CLI 用户：** 双击 `3-update.bat`（Windows）或运行 `./3-update.sh`（macOS/Linux）。这会通过 git 拉取最新代码；如果 git 不可用，则下载 ZIP。

> **注意：** GitHub Releases 上的更新 ZIP（`EZ-CorridorKey-windows-x64.zip`）仅面向 Windows 桌面应用安装程序用户。它用于修补现有安装。CLI 用户应继续使用 `3-update.bat` / `3-update.sh`。

部分可选模型（BiRefNet、SAM2）通过 Hugging Face Hub 下载，并缓存在项目文件夹外。要回收磁盘空间：

☼ **Windows：** `%USERPROFILE%\.cache\huggingface\hub\`

## 快速开始

### 1. 导入

将视频文件拖到欢迎屏幕上（或文件 > 导入片段 > 导入视频）。视频会自动提取为高质量图像序列：

1. **FFmpeg** 将每一帧提取为 **EXR half-float**（ZIP16 压缩）
2. **重压缩步骤** 将 ZIP16 → **DWAB**（VFX 标准有损压缩，约小 4 倍）

这种两步方式会保留来自视频解码器的完整浮点精度，没有 8 位量化，也没有色带。即使源视频是 8 位，也能受益，因为 FFmpeg 内部 YUV→RGB 转换保持在 float 中，避免整数流程中累积的舍入误差。DWAB 压缩的 EXR 帧通常接近 PNG 大小，同时保留 16 位 half-float 动态范围。

重压缩在单独进程中运行，因此提取期间 UI 仍能完全响应。

### 2. 生成 Alpha 提示

你的片段从 **RAW** 状态开始（灰色徽章）。运行推理前需要 Alpha 提示。

**选项 A：色度键（手动，无 AI 模型）：**
点击参数面板中的 **CHROMA KEY**。这是用于绿幕或蓝幕素材的色差键控器。按 **E** 激活吸管，然后在画面上拖动以采样一段颜色范围。调整 Key Strength、Clip Black 和 Clip White 来细化遮罩。点击 **GENERATE** 生成 Alpha 提示。

**选项 B：一键 Alpha 生成器：**
☼ **GVM Auto**：点击参数面板中的 **GVM AUTO**。对大多数有人物的绿幕素材效果很好。

☼ **BiRefNet**（推荐）：点击参数面板中的 **BIREFNET**。速度快、准确，并适用于多种素材。


**选项 C：MatAnyone2 / VideoMaMa：**
这些模型需要蒙版提示。可通过两种方式提供：

*C1. 绘制 + 跟踪（从零开始）：*
1. 按 **1** 激活前景模式（绿色）
2. 在几个关键帧上涂抹主体
3. 按 **2** 切换到背景模式（红色）
4. 在背景区域涂抹
5. 点击 **TRACK MASK** 生成密集 SAM2 蒙版跟踪
6. 点击参数面板中的 **MATANYONE2** 或 **VIDEOMAMA**

*C2. 导入蒙版（自备）：*
1. 点击 **VIDEOMAMA** 旁边的 **+** 按钮，导入预制蒙版序列或视频
2. 点击 **MATANYONE2** 或 **VIDEOMAMA** 运行模型

**选项 D：导入 Alpha（自备）：**
如果你已经从其他工具（Rotobrush、Silhouette、Resolve、Nuke 等）获得 Alpha 遮罩，请点击参数面板中的 **IMPORT ALPHA**，并选择图像文件夹或遮罩视频文件。

☼ 支持的图像格式：**PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ 支持的 Alpha 视频路径：普通片段导入器接受的标准视频文件（例如 **MOV** 或 **MP4**）

☼ 图像和可见遮罩视频应为 **grayscale**（白色 = 前景，黑色 = 背景）

☼ 帧数应与输入序列匹配

☼ 导入时，非 PNG 静帧会自动转换为 grayscale PNG

☼ 导入的 Alpha 视频会解码为 grayscale `AlphaHint/*.png` 帧，因此与图像序列提示使用相同的后续路径

☼ 导入的文件会复制到片段的 `AlphaHint/` 文件夹，片段进入 **READY** 状态

你可以随时重新导入。如果片段已有 Alpha 提示，系统会询问是否覆盖。

### 3. 运行推理

你的片段现在是 **READY**（黄色徽章）。按需调整参数，然后点击 **RUN INFERENCE**（或 **Ctrl+R**）。

### 批处理流程（无人值守）

在 I/O 托盘中选择多个片段（Ctrl+点击或 Shift+点击），然后点击状态栏中的 **RUN PIPELINE**。系统会自动：

1. 检测每个片段的状态和绘制笔触
2. 对已绘制片段运行 **TRACK MASK**，然后运行 **VideoMaMa**
3. 对未绘制的 RAW 片段运行 **GVM Auto**
4. 每次 Alpha 生成完成后串联运行**推理**

整个流程可取消（Esc）且可断点续跑。若中断，重新启动会从每个片段上次停止的位置继续。

### 4. 检查

在不同查看模式之间切换以检查结果：

☼ **COMP**：在棋盘格上查看抠像

☼ **FG**：检查绿色边缘污染

☼ **MATTE**：检查 Alpha 质量

☼ **PROCESSED**：制作级 RGBA

推理期间，输出会写入项目的 `Output/` 子目录（可配置，见[自定义输出目录](#自定义输出目录)）。

### Windows / Linux (NVIDIA CUDA)

| -------- | ------------- | ------------------ | ----------------------------- |

#### VRAM 模式

| 模式                  | VRAM 使用量 | 4K 速度    | 工作方式                                 |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | 全帧精修器，完整 torch.compile   |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | 512×512 平铺精修器，选择性编译 |

模式会根据可用 VRAM 自动检测。可用 `CORRIDORKEY_OPT_MODE=speed|lowvram|auto` 覆盖。

### macOS (Apple Silicon)

> 适用于 v2.0.0 和源码安装。像 2.1.4 这样的补丁版本以应用内自动更新的形式发布，不提供新的 Mac 安装包。

|             | 最低          | 推荐                                             |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX（安装 corridorkey-mlx 时自动检测） |

CorridorKey 推理会自动选择最快可用后端：当存在 `corridorkey-mlx` 和 `.safetensors` checkpoint 时使用 MLX（快 1.5–2 倍），否则使用 PyTorch MPS。可用 `CORRIDORKEY_BACKEND=torch|mlx|auto` 覆盖。

Alpha 生成器（SAM2、GVM、VideoMaMa、MatAnyone2）始终在 PyTorch MPS 上运行。这些模型没有 MLX 移植。为了获得最佳 Mac 体验，请导入来自 After Effects、DaVinci Resolve 或 Nuke 的预制 Alpha 遮罩。
