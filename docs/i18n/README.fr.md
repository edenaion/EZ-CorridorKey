Une interface graphique de bureau complète pour [CorridorKey de Niko Pueringer](https://github.com/nikopueringer/CorridorKey), le chroma keyer IA de Corridor Digital qui démixe physiquement le premier plan de l'arrière-plan, tout en préservant les cheveux, le flou de mouvement et la translucidité.

Cette interface remplace le flux CLI par glisser-déposer par une application de bureau complète, tout en conservant une compatibilité arrière à 100 % (`python main.py --cli` lance toujours l'assistant d'origine).

## Installation

### Installateur d'application de bureau (recommandé)

**Vous ne voulez pas gérer Python, git ou la ligne de commande ?** Un installateur Windows complet, un exe portable et un `.pkg` macOS sont disponibles. Ils sont entièrement optionnels et gratuits. Les dons soutiennent le développement actif.

[![Télécharger sur Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

L'installateur inclut tout : runtime Python, modèles IA, bibliothèques GPU. Aucune configuration requise. Installez puis lancez.

### Installation par terminal (CLI) (Windows / macOS / Linux)

1. Clonez ou téléchargez ce dépôt.
2. Le parcours en un clic provisionne et utilise automatiquement Python 3.11 géré. Vous n'avez donc pas besoin de préinstaller Python juste pour utiliser `1-install`.
3. Lancez l'installateur pour votre plateforme :
   ☼ **Windows :** Double-cliquez sur `1-install.bat`

   ☼ **macOS / Linux :** `chmod +x 1-install.sh && ./1-install.sh`

4. L'installateur gère tout : Python géré, environnement virtuel, dépendances (dont le bon backend PyTorch pour votre GPU quand il est disponible), vérification et téléchargement des modèles.
5. Pour lancer : double-cliquez sur `2-start.bat` (Windows) ou `./2-start.sh` (macOS/Linux).

**Prérequis :**

☼ Pour l'installateur en un clic : aucun Python préinstallé requis

☼ Pour les installations manuelles : [Python 3.10–3.13](https://python.org) (3.14 n'est pas encore pris en charge)

☼ **Windows/Linux :** GPU NVIDIA avec prise en charge CUDA (8 Go+ de VRAM recommandés). Gardez votre pilote à jour. L'installateur vérifie le runtime torch et s'arrête avec des diagnostics au lieu de vous laisser silencieusement sur le mauvais backend.

☼ **macOS :** Apple Silicon (M1+). L'inférence CorridorKey s'exécute nativement via MLX (1,5 à 2x plus rapide que MPS). Les générateurs d'alpha lourds pour le GPU (SAM2, GVM, VideoMaMa, MatAnyone2) s'exécutent sur MPS mais sont nettement plus lents. L'import de mattes alpha déjà créées est recommandé sur Mac.

**Ce que fait l'installateur :**

☼ Vérifie [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (compilateur C++ requis par OpenEXR) et propose l'installation automatique s'il manque

☼ Installe [uv](https://docs.astral.sh/uv/) et provisionne Python 3.11 géré pour le chemin de l'installateur

☼ Crée un environnement virtuel `.venv` dans le dossier du projet

☼ Installe le backend PyTorch correct pour votre plateforme/GPU et vérifie le runtime torch obtenu avant d'annoncer la réussite

☼ Télécharge et installe [FFmpeg](https://ffmpeg.org/) localement s'il est absent du PATH (utilisé pour l'import vidéo)

☼ Télécharge le checkpoint du modèle CorridorKey (383 Mo, requis)

☼ Installe en option la prise en charge du suivi SAM2 et pré-télécharge le checkpoint Base+ par défaut (324 Mo)

☼ Télécharge en option les générateurs d'alpha BiRefNet (~940 Mo), MatAnyone2 (~135 Mo), GVM (~6 Go) et VideoMaMa (~37 Go)

☼ Crée un raccourci sur le bureau (optionnel)

**Mise à jour :**

☼ **Utilisateurs de l'installateur Windows Desktop App :** L'application vérifie les mises à jour automatiquement. Lorsqu'une nouvelle version est disponible, cliquez sur le bouton de mise à jour dans l'app. Elle télécharge un patch léger et relance l'application.

☼ **Utilisateurs de l'application macOS Desktop App en 1.9.x :** 1.10.0 ne se met **pas** à jour automatiquement sur Mac. Vous devez télécharger le nouveau `.pkg` depuis Gumroad et l'exécuter manuellement. L'installateur remplace l'application existante `/Applications/EZ-CorridorKey.app` en place. Vos préférences, projets et modèles téléchargés se trouvent dans `~/Library/Application Support/EZ-CorridorKey/` et sont conservés pendant la réinstallation. La mise à jour automatique Mac revient dans 1.11.0 ou une version ultérieure.

☼ **Utilisateurs CLI :** Double-cliquez sur `3-update.bat` (Windows) ou lancez `./3-update.sh` (macOS/Linux). Cela récupère le dernier code via git ou télécharge un ZIP si git n'est pas disponible.

> **Note :** Le ZIP de mise à jour sur GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) est réservé aux utilisateurs de l'installateur Windows Desktop App. Il patche une installation existante. Les utilisateurs CLI doivent continuer à utiliser `3-update.bat` / `3-update.sh`.

### Installation alternative : Docker

Pour les utilisateurs Linux ou les configurations distantes/cloud, EZ-CorridorKey peut s'exécuter dans Docker avec un accès par navigateur via noVNC. Consultez [docker/README.md](docker/README.md) pour les instructions de configuration. L'installation native ci-dessus est recommandée pour Windows et macOS.

## Désinstallation

### Installateur d'application de bureau

**Windows :**
☼ Ouvrez Settings > Apps > Installed Apps

☼ Trouvez **EZ-CorridorKey** puis cliquez sur Uninstall

☼ Cela supprime l'application et le raccourci du menu Démarrer

☼ Vos projets et modèles téléchargés sont stockés dans `%APPDATA%\EZ-CorridorKey\`. Supprimez ce dossier pour retirer toutes les données utilisateur.

**macOS :**
☼ Faites glisser `/Applications/EZ-CorridorKey.app` vers la corbeille

☼ Vos projets, préférences et modèles téléchargés sont stockés dans `~/Library/Application Support/EZ-CorridorKey/`. Supprimez ce dossier pour retirer toutes les données utilisateur.

### Installation CLI (git clone)

☼ Supprimez le dossier du dépôt cloné (par exemple `EZ-CorridorKey/`). Cela inclut l'environnement virtuel `.venv`, les modèles téléchargés et toutes les données de projet dans `Projects/`.

☼ Si vous avez créé un raccourci bureau pendant l'installation, supprimez-le manuellement.

☼ Aucun fichier système n'est modifié par l'installation CLI. Il n'y a rien d'autre à nettoyer.

### Cache de modèles Hugging Face

Certains modèles optionnels (BiRefNet, SAM2) sont téléchargés via Hugging Face Hub et mis en cache hors du dossier du projet. Pour récupérer cet espace disque :

☼ **Windows :** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux :** `~/.cache/huggingface/hub/`

Ce cache est partagé par toutes les applications qui utilisent Hugging Face. Si vous utilisez d'autres outils IA, supprimez seulement les dossiers de modèles concernés (par exemple `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`) plutôt que tout le répertoire `hub/`.

## Démarrage rapide

### 1. Importer

Déposez un fichier vidéo sur l'écran d'accueil (ou File > Import Clips > Import Video). La vidéo est automatiquement extraite en séquence d'images haute qualité :

1. **FFmpeg extrait** chaque image en **EXR demi-précision** (compression ZIP16)
2. **Passe de recompression** : ZIP16 → **DWAB** (compression avec perte standard VFX, ~4× plus petit)

Cette approche en deux passes préserve toute la précision en virgule flottante du décodeur vidéo : pas de quantification 8 bits, pas de banding. Même les vidéos source 8 bits en bénéficient, car la conversion interne YUV→RGB de FFmpeg reste en float et évite les erreurs d'arrondi qui s'accumulent dans les pipelines entiers. Les images EXR compressées en DWAB ont généralement une taille comparable au PNG tout en conservant une plage dynamique half-float 16 bits.

La recompression s'exécute dans un processus séparé afin que l'interface reste pleinement réactive pendant l'extraction.

### 2. Générer l'alpha de guidage

Votre clip démarre à l'état **RAW** (badge gris). Vous avez besoin d'un alpha de guidage avant de lancer l'inférence.

**Option A - Chroma Key (manuel, sans modèle IA) :**
Cliquez sur **CHROMA KEY** dans le panneau de paramètres. Keyer par différence de couleur pour les plans sur fond vert ou bleu. Appuyez sur **E** pour activer la pipette, puis faites glisser sur l'écran pour échantillonner une plage de couleurs. Ajustez Key Strength, Clip Black et Clip White pour affiner le matte. Cliquez sur **GENERATE** pour produire l'alpha de guidage.

**Option B - Générateurs d'alpha en un clic :**
☼ **GVM Auto** - cliquez sur **GVM AUTO** dans le panneau de paramètres. Fonctionne très bien pour la plupart des plans fond vert avec des personnes.

☼ **BiRefNet** (recommandé) - cliquez sur **BIREFNET** dans le panneau de paramètres. Rapide, précis et adapté à une large gamme de plans.

**Option C - MatAnyone2 / VideoMaMa :**
Ces modèles ont besoin d'un guide de masque. Deux méthodes :

*C1. Peinture + suivi (depuis zéro) :*
1. Appuyez sur **1** pour activer le mode premier plan (vert)
2. Peignez le sujet sur quelques images clés
3. Appuyez sur **2** pour passer au mode arrière-plan (rouge)
4. Peignez les zones d'arrière-plan
5. Cliquez sur **TRACK MASK** pour générer une piste de masque SAM2 dense
6. Cliquez sur **MATANYONE2** ou **VIDEOMAMA** dans le panneau de paramètres

*C2. Importer un masque (fourni par vous) :*
1. Cliquez sur le bouton **+** à côté de **VIDEOMAMA** pour importer une séquence de masques ou une vidéo déjà créée
2. Cliquez sur **MATANYONE2** ou **VIDEOMAMA** pour lancer le modèle

**Option D - Importer l'alpha (fourni par vous) :**
Si vous avez déjà des mattes alpha depuis un autre outil (Rotobrush, Silhouette, Resolve, Nuke, etc.), cliquez sur **IMPORT ALPHA** dans le panneau de paramètres et choisissez un dossier d'images ou un fichier vidéo de matte.

☼ Formats d'image pris en charge : **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Chemin vidéo alpha pris en charge : fichiers vidéo standard acceptés par l'importateur de clips normal (par exemple **MOV** ou **MP4**)

☼ Les images et vidéos matte visibles doivent être en **niveaux de gris** (blanc = premier plan, noir = arrière-plan)

☼ Le nombre d'images doit correspondre à votre séquence d'entrée

☼ Les images fixes non PNG sont automatiquement converties en PNG niveaux de gris lors de l'import

☼ Les vidéos alpha importées sont décodées en images niveaux de gris `AlphaHint/*.png` afin de suivre le même chemin aval que les guides en séquence d'images

☼ Les fichiers importés sont copiés dans le dossier `AlphaHint/` du clip et le clip passe à l'état **READY**

Vous pouvez réimporter à tout moment. Si le clip possède déjà des guides alpha, l'application vous demandera si vous voulez les remplacer.

### 3. Lancer l'inférence

Votre clip est maintenant **READY** (badge jaune). Ajustez les paramètres si nécessaire, puis cliquez sur **RUN INFERENCE** (ou **Ctrl+R**).

### Pipeline par lots (sans surveillance)

Sélectionnez plusieurs clips dans le tiroir I/O (Ctrl+clic ou Shift+clic), puis cliquez sur **RUN PIPELINE** dans la barre d'état. Le système :

1. Détecte l'état et les traits de peinture de chaque clip
2. Lance **TRACK MASK** puis **VideoMaMa** pour les clips peints
3. Lance **GVM Auto** pour les clips RAW non peints
4. Enchaîne **inference** après chaque génération d'alpha terminée

Tout le pipeline est annulable (Esc) et checkpointable. S'il est interrompu, le redémarrage reprend là où chaque clip s'est arrêté.

### 4. Revoir

Basculez entre les modes d'affichage pour inspecter les résultats :

☼ **COMP** - key sur damier

☼ **FG** - vérifier le fringing vert

☼ **MATTE** - inspecter la qualité de l'alpha

☼ **PROCESSED** - RGBA de production

Les sorties sont écrites dans les sous-répertoires `Output/` du projet pendant l'inférence (configurable, voir [Dossier de sortie personnalisé](#dossier-de-sortie-personnalisé)).

## Configuration matérielle requise

### Windows / Linux (NVIDIA CUDA)

|          | Minimum       | Recommandé         | Confortable                   |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 Go          | 12 Go              | 16 Go+                        |
| **GPU**  | NVIDIA (CUDA) | Ampere+ (RTX 30xx) | Ada/Blackwell (RTX 40xx/50xx) |

#### Modes VRAM

| Mode                  | Utilisation VRAM | Vitesse 4K | Fonctionnement                           |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Refiner pleine image, torch.compile complet |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | Refiner tuilé 512×512, compilation sélective |

Le mode est détecté automatiquement depuis la VRAM disponible. Remplacez avec `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

|             | Minimum       | Recommandé                                       |
| ----------- | ------------- | ------------------------------------------------ |
| **Puce**    | M1 (8 Go)     | M1 Pro+ (16 Go+)                                 |
| **Backend** | MPS (PyTorch) | MLX (auto-détecté si corridorkey-mlx est installé) |

L'inférence CorridorKey sélectionne automatiquement le backend le plus rapide disponible : MLX (1,5 à 2x plus rapide) quand `corridorkey-mlx` et un checkpoint `.safetensors` sont présents, sinon PyTorch MPS. Remplacez avec `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Les générateurs d'alpha (SAM2, GVM, VideoMaMa, MatAnyone2) tournent toujours sur PyTorch MPS. Il n'existe pas de ports MLX pour ces modèles. Pour la meilleure expérience sur Mac, importez des mattes alpha déjà créées depuis After Effects, DaVinci Resolve ou Nuke.

> **Problème connu :** La sortie FG MLX peut produire des résultats en blocs ou avec artefacts. Désactivez la sortie FG et utilisez plutôt les sorties Matte + Processed. Les utilisateurs CUDA ne sont pas affectés.

### Mises à jour signées

À partir de v1.10.0, chaque release GitHub contient un manifeste signé (`manifest.json` + `manifest.json.sig`). Quand le programme de mise à jour intégré télécharge une nouvelle version, il vérifie la signature Ed25519 et le hash SHA-256 avant d'appliquer la mise à jour. Si l'une des vérifications échoue, la mise à jour est rejetée et l'utilisateur est averti.

Cela signifie que même si le compte GitHub était compromis, l'attaquant ne pourrait pas pousser une mise à jour malveillante aux utilisateurs installés sans posséder aussi la clé de signature hors ligne.

Le code de vérification est open source : [`backend/update_verify.py`](backend/update_verify.py).

### Sommes de contrôle

Chaque release inclut un fichier `SHA256SUMS.txt` qui liste le hash SHA-256 de chaque artefact de release. Téléchargez-le avec le binaire et vérifiez manuellement :

```bash
# macOS / Linux
sha256sum -c SHA256SUMS.txt

# Windows PowerShell
(Get-FileHash EZ-CorridorKey-2.0.0-Windows-x64-Setup.exe -Algorithm SHA256).Hash
# Compare the output against the matching line in SHA256SUMS.txt
```

### Intégrité Git

Tous les commits vers `main` sont signés avec une clé SSH Ed25519. La protection de branche impose les commits signés, les revues de pull request, et bloque les force pushes et la suppression de branche.

## Localisation

EZ-CorridorKey prend en charge la traduction dans n'importe quelle langue. Toutes les chaînes de l'interface sont extraites dans des fichiers Qt `.ts` standard que les traducteurs peuvent modifier avec [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (gratuit) ou n'importe quel éditeur de texte.

**Comment ajouter votre langue :**

1. Copiez `ui/translations/corridorkey_en.ts` vers votre code de langue (par exemple `corridorkey_fr.ts` pour le français)
2. Ouvrez le nouveau fichier dans Qt Linguist et remplissez les traductions
3. Compilez-le : `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Soumettez une pull request avec les fichiers `.ts` et `.qm`

Les instructions complètes avec exemples se trouvent dans [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Langues actuelles :** Anglais (intégré). Vous voulez voir votre langue ici ? Les PR sont bienvenues.
