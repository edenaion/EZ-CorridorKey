## 安裝

**偏好影片教學？**觀看應用程式作者的[逐步教學影片](https://www.youtube.com/watch?v=ezg_1oQzz7w)：

### 桌面應用程式安裝程式（推薦）

**不想處理 Python、git 或命令列？** 提供完整的 Windows 安裝程式與可攜式 exe。Mac 使用者可下載 v2.0.0 的 macOS `.pkg`，同時原生 Mac 應用程式正在開發中。完全免費自由選用，捐款可支持持續開發。

[![在 Gumroad 下載](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

安裝程式包含所有內容：Python 執行環境、AI 模型、GPU 函式庫，無需額外設定。安裝後即可直接執行。

### 終端機（CLI）安裝（Windows / macOS / Linux）

1. 複製或下載此儲存庫。
2. 一鍵路徑會自動準備並使用受管理的 Python 3.11，因此您無需預先安裝 Python 即可使用 `1-install`。
3. 執行適合您平台的安裝程式：
   - **Windows：** 雙擊 `1-install.bat`
   - **macOS / Linux：** `chmod +x 1-install.sh && ./1-install.sh`
4. 安裝程式會處理所有項目：受管理的 Python、虛擬環境、相依套件（包含在可用時為您的 GPU 安裝正確的 PyTorch 後端）、驗證及模型下載。
5. 啟動方式：雙擊 `2-start.bat`（Windows）或執行 `./2-start.sh`（macOS/Linux）。

**先決條件：**

- 一鍵安裝程式：無需預先安裝 Python
- 手動安裝：[Python 3.10–3.13](https://python.org)（尚不支援 3.14）
- **Windows/Linux：** 支援 CUDA 的 NVIDIA GPU（建議 8 GB 以上 VRAM）。請保持驅動程式為最新版本。安裝程式會驗證 torch 執行環境，若後端有誤將提供診斷資訊並停止，而不會靜默留下錯誤設定。
- **macOS：** Apple Silicon（M1+）。CorridorKey 推理透過 MLX 原生執行（比 MPS 快 1.5 至 2 倍）。GPU 密集型 Alpha 產生器（SAM2、GVM、VideoMaMa、MatAnyone2）在 MPS 上執行，但速度明顯較慢，Mac 上建議匯入預製的 Alpha 遮罩。

**安裝程式執行內容：**

- 檢查 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)（OpenEXR 需要 C++ 編譯器），若未安裝則提供自動安裝選項
- 安裝 [uv](https://docs.astral.sh/uv/) 並為安裝路徑準備受管理的 Python 3.11
- 在專案資料夾中建立 `.venv` 虛擬環境
- 為您的平台／GPU 安裝正確的 PyTorch 後端，並在回報成功前驗證所得的 torch 執行環境
- 若 PATH 中找不到 [FFmpeg](https://ffmpeg.org/)，則在本機下載並安裝（用於影片匯入）
- 下載 CorridorKey 模型檢查點（383 MB，必要）
- 選擇性安裝 SAM2 追蹤支援，並預先下載預設 Base+ 檢查點（324 MB）
- 選擇性下載 BiRefNet（約 940 MB）、MatAnyone2（約 135 MB）、GVM（約 6 GB）及 VideoMaMa（約 37 GB）Alpha 提示產生器
- 建立桌面捷徑（選擇性）

**更新：**

- **Windows 桌面應用程式安裝程式使用者：** 應用程式會自動檢查更新。當新版本可用時，點擊應用程式內的更新按鈕。它會下載輕量修補程式並重新啟動。
- **macOS 桌面應用程式使用者：** 應用程式會自動檢查更新。當新版本可用時，點擊應用程式內的更新按鈕。它會下載輕量修補程式並重新啟動。
- **CLI 使用者：** 雙擊 `3-update.bat`（Windows）或執行 `./3-update.sh`（macOS/Linux）。此操作會透過 git 拉取最新程式碼；若 git 不可用，則下載 ZIP。

> **注意：** GitHub Releases 上的更新 ZIP（`EZ-CorridorKey-windows-x64.zip`）僅供 Windows 桌面應用程式安裝程式使用者使用，用於修補現有安裝。CLI 使用者應繼續使用 `3-update.bat` / `3-update.sh`。

### 替代安裝：Docker

Linux 使用者或遠端／雲端環境，可在 Docker 中執行 EZ-CorridorKey，並透過 noVNC 以瀏覽器存取。設定說明請參閱 [docker/README.md](docker/README.md)。Windows 與 macOS 建議使用上述原生安裝方式。

## 解除安裝

### 桌面應用程式安裝程式

**Windows：**
☼ 開啟設定 > 應用程式 > 已安裝的應用程式
☼ 找到 **EZ-CorridorKey** 並點擊解除安裝
☼ 這將移除應用程式與開始功能表捷徑
☼ 您的專案與已下載模型儲存在 `%APPDATA%\EZ-CorridorKey\`。刪除該資料夾即可移除所有使用者資料。

**macOS：**
☼ 將 `/Applications/EZ-CorridorKey.app` 拖曳至垃圾桶
☼ 您的專案、偏好設定與已下載模型儲存在 `~/Library/Application Support/EZ-CorridorKey/`。刪除該資料夾即可移除所有使用者資料。

### CLI 安裝（git clone）

☼ 刪除已複製的儲存庫資料夾（例如 `EZ-CorridorKey/`）。其中包含 `.venv` 虛擬環境、已下載模型，以及 `Projects/` 內的所有專案資料。
☼ 若安裝時建立了桌面捷徑，請手動刪除。
☼ CLI 安裝不會修改系統層級檔案，無需進行其他清理。

### Hugging Face 模型快取

部分選用模型（BiRefNet、SAM2）透過 Hugging Face Hub 下載，並快取在專案資料夾外。若要回收磁碟空間：

☼ **Windows：** `%USERPROFILE%\.cache\huggingface\hub\`
☼ **macOS / Linux：** `~/.cache/huggingface/hub/`

此快取由所有使用 Hugging Face 的應用程式共用。若您同時使用其他 AI 工具，請僅刪除特定模型資料夾（例如 `models--ZhengPeng7--BiRefNet`、`models--facebook--sam2.1-hiera-base-plus`），而非整個 `hub/` 目錄。

## 快速開始

### 1. 匯入

將影片檔案拖放至歡迎畫面（或選擇檔案 > 匯入片段 > 匯入影片）。影片會自動擷取為高品質影像序列：

1. **FFmpeg 擷取**每個影格為 **EXR half-float**（ZIP16 壓縮）
2. **重新壓縮程序**將 ZIP16 轉換為 **DWAB**（VFX 標準有損壓縮，縮小約 4 倍）

這種兩步驟方式可保留來自影片解碼器的完整浮點精度，不會有 8 位元量化或色帶問題。即使是 8 位元來源影片也能受益，因為 FFmpeg 內部的 YUV→RGB 轉換保持在浮點數中，避免了整數處理流程中累積的捨入誤差。DWAB 壓縮的 EXR 影格通常與 PNG 大小相當，同時保留 16 位元 half-float 動態範圍。

重新壓縮在獨立程序中執行，因此擷取期間 UI 仍可完全回應。

**配對提示：** 檔名中任何位置含有「alphahint」或「maskhint」的檔案（不區分大小寫）會自動與對應片段配對，作為 Alpha 或遮罩提示，而非作為獨立片段匯入。適用於影片匯入與影格序列資料夾。

### 2. 產生 Alpha 提示

您的片段從 **RAW** 狀態開始（灰色徽章）。執行推理前需要 Alpha 提示。

**選項 A — 色鍵（手動，無需 AI 模型）：**
點擊參數面板中的 **CHROMA KEY**。這是用於綠幕或藍幕素材的色差鍵控器。按 **E** 啟用滴管，然後在畫面上拖曳以取樣一段色彩範圍。調整 Key Strength、Clip Black 與 Clip White 來精修遮罩。點擊 **GENERATE** 產生 Alpha 提示。

**選項 B — 一鍵 Alpha 產生器：**
- **GVM Auto** — 點擊參數面板中的 **GVM AUTO**。對大多數含有人物的綠幕素材效果極佳。
- **BiRefNet**（推薦）— 點擊參數面板中的 **BIREFNET**。速度快、準確，且適用於多種素材類型。
- **Apple Vision**（macOS 14+）— 點擊參數面板中的 **APPLE VISION**。Apple 神經引擎分割前景，無需繪製或標記。在其他平台上自動隱藏。

**選項 C — MatAnyone2 / VideoMaMa：**
這些模型需要遮罩提示。可透過以下兩種方式提供：

*C1. 繪製 + 追蹤（從頭開始）：*
1. 按 **1** 啟用前景模式（綠色）
2. 在幾個關鍵影格上塗抹主體
3. 按 **2** 切換至背景模式（紅色）
4. 在背景區域塗抹
5. 點擊 **TRACK MASK** 產生密集的 SAM2 遮罩追蹤
6. 點擊參數面板中的 **MATANYONE2** 或 **VIDEOMAMA**

*C2. 匯入遮罩（自備）：*
1. 點擊 **VIDEOMAMA** 旁的 **+** 按鈕，匯入預製遮罩序列或影片
2. 點擊 **MATANYONE2** 或 **VIDEOMAMA** 執行模型

**選項 D — 匯入 Alpha（自備）：**
若您已從其他工具（Rotobrush、Silhouette、Resolve、Nuke 等）取得 Alpha 遮罩，請點擊參數面板中的 **IMPORT ALPHA**，並選擇影像資料夾或遮罩影片檔案。

- 支援的影像格式：**PNG、JPG、JPEG、TIF、TIFF、EXR**
- 支援的 Alpha 影片路徑：一般片段匯入器接受的標準影片檔案（例如 **MOV** 或 **MP4**）
- 影像與可見遮罩影片應為**灰階**（白色 = 前景，黑色 = 背景）
- 影格數應與輸入序列相符
- 非 PNG 靜態影像在匯入時會自動轉換為灰階 PNG
- 匯入的 Alpha 影片會解碼為灰階 `AlphaHint/*.png` 影格，因此與影像序列提示使用相同的後續處理路徑
- 匯入的檔案會複製到片段的 `AlphaHint/` 資料夾，片段進入 **READY** 狀態

您可以隨時重新匯入。若片段已有 Alpha 提示，系統會詢問是否覆蓋。

### 3. 執行推理

您的片段現在是 **READY**（黃色徽章）。依需求調整參數後，點擊 **RUN INFERENCE**（或 **Ctrl+R**）。

### 批次流程（無人值守）

執行批次的兩種方式：

**從 I/O 托盤：** 選取多個片段（Ctrl+點擊或 Shift+點擊），然後點擊狀態列中的 **RUN PIPELINE**。

**從資料夾：** 檔案 > 批次流程會開啟對話框，用於處理整個資料夾的片段。選擇資料夾，選擇要使用的 Alpha 產生模型（GVM、BiRefNet、VideoMaMa、MatAnyone2），然後全部自動執行。每個片段的個別覆蓋設定允許在同一批次中混用模型。即時進度列與核取符號追蹤每個片段的狀態。影像序列子資料夾會被偵測到，可與影片在同一批次中混合使用。

對於托盤中選取的片段，系統會自動：

1. 偵測每個片段的狀態與繪製筆觸
2. 對已繪製的片段執行 **TRACK MASK**，然後執行 **VideoMaMa**
3. 對未繪製的 RAW 片段執行 **GVM Auto**
4. 每次 Alpha 產生完成後串聯執行**推理**

整個流程可取消（Esc）且支援中斷續跑。若中斷，重新啟動會從每個片段上次停止的位置繼續。

### 4. 審閱

在不同檢視模式間切換以檢查結果：

- **COMP** — 在棋盤格上查看去背效果
- **FG** — 檢查綠色邊緣污染
- **MATTE** — 檢查 Alpha 品質
- **PROCESSED** — 製作級 RGBA

推理期間，輸出會寫入專案的 `Output/` 子目錄（可設定，請參閱[自訂輸出目錄](#自訂輸出目錄)）。

**色彩空間行為**

- **左側 INPUT 檢視器**始終顯示 CorridorKey 目前對來源素材的解譯。若輸入在此看起來有誤，未來的推理結果與匯出也將基於這個錯誤解譯。
- 在點擊 **RUN INFERENCE** 前變更 **Color Space**，會影響下一次即時預覽與下一次匯出執行的產生方式。
- 輸出已存在後再變更 **Color Space**，**不會**重新寫入磁碟上的檔案。它只會更新檢視器與即時預覽。若要將新解譯儲存至檔案，請重新執行推理。

### macOS（Apple Silicon）

> 適用於 v2.0.0 與原始碼安裝。像 2.1.4 這樣的修補版本以應用程式內自動更新的形式發佈，不提供新的 Mac 安裝程式。

|             | 最低配置      | 建議配置                                              |
| ----------- | ------------- | ----------------------------------------------------- |
| **晶片**    | M1（8 GB）    | M1 Pro+（16 GB+）                                     |
| **後端**    | MPS (PyTorch) | MLX（內建，自動偵測）                                 |

## 隱私權

EZ-CorridorKey 預設不傳送任何遙測資料。沒有使用分析，沒有追蹤，永遠如此。

以下兩項功能可能會傳送資料，且均由您自行控制：

☼ **回報問題**：當您透過說明選單提交回報時，您在對話框中看到的診斷摘要也可選擇直接傳送給開發者。對話框中有一個可見的核取方塊，可針對每次回報個別控制。
☼ **當機回報**：偏好設定 > 隱私權中的選用切換，預設為關閉。啟用後，當機詳細資訊將自動傳送。

回報內容包含：當機詳情、GPU/驅動程式資訊、作業系統與應用程式版本。絕不包含：您的媒體檔案、檔案名稱、專案或個人資料。傳送前，檔案路徑會進行匿名化處理。

## 在地化

EZ-CorridorKey 支援翻譯為任何語言。所有 UI 字串均擷取至標準 Qt `.ts` 檔案，翻譯者可使用 [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html)（免費）或任何文字編輯器進行編輯。

**如何新增您的語言：**

1. 將 `ui/translations/corridorkey_en.ts` 複製為您的語言代碼（例如法語為 `corridorkey_fr.ts`）
2. 在 Qt Linguist 中開啟新檔案並填入翻譯
3. 編譯：`pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. 提交包含 `.ts` 與 `.qm` 檔案的 pull request

完整說明與範例請參閱 [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md)。

**目前語言：** 英語，加上德語、西班牙語、法語、印地語、印尼語、義大利語、日語、韓語、波蘭語、葡萄牙語（巴西）、俄語、土耳其語、烏克蘭語、越南語、簡體中文與繁體中文（台灣）。請在編輯 > 偏好設定中選擇您的語言。希望在此看到您的語言，或想改善現有翻譯？歡迎提交 PR。
