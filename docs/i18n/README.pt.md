Uma GUI desktop completa para o [CorridorKey de Niko Pueringer](https://github.com/nikopueringer/CorridorKey), o chroma keyer por IA da Corridor Digital que separa fisicamente o primeiro plano do fundo, preservando cabelo, motion blur e translucidez.

## Instalação

**Prefere vídeo?** Assista ao [tutorial passo a passo](https://www.youtube.com/watch?v=ezg_1oQzz7w) do criador do app:

<a href="https://www.youtube.com/watch?v=ezg_1oQzz7w"><img src="https://img.youtube.com/vi/ezg_1oQzz7w/maxresdefault.jpg" alt="EZ-CorridorKey video tutorial" width="640"></a>

### Instalador do app desktop (recomendado)

**Não quer lidar com Python, git ou linha de comando?** Há um instalador Windows completo, um exe portátil e um `.pkg` para macOS. São totalmente opcionais e gratuitos. Doações ajudam a manter o desenvolvimento ativo.

[![Baixar no Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

O instalador inclui tudo: runtime Python, modelos de IA e bibliotecas de GPU. Nenhuma configuração necessária. Basta instalar e executar.

### Instalação pelo terminal (CLI) (Windows / macOS / Linux)

1. Clone ou baixe este repositório.
2. O caminho one-click provisiona e usa Python 3.11 gerenciado automaticamente, então você não precisa pré-instalar Python só para usar `1-install`.
3. Execute o instalador da sua plataforma:
   ☼ **Windows:** clique duas vezes em `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. O instalador cuida de tudo: Python gerenciado, ambiente virtual, dependências (incluindo o backend PyTorch correto para sua GPU quando disponível), verificação e downloads de modelos.
5. Para iniciar: clique duas vezes em `2-start.bat` (Windows) ou execute `./2-start.sh` (macOS/Linux).

**Pré-requisitos:**

☼ Para o instalador one-click: Python pré-instalado não é necessário

☼ Para instalações manuais: [Python 3.10-3.13](https://python.org) (3.14 ainda não é suportado)

☼ **Windows/Linux:** GPU NVIDIA com suporte a CUDA (8 GB+ de VRAM recomendado). Mantenha o driver atualizado. O instalador verifica o runtime torch e para com diagnóstico em vez de deixar você no backend errado sem aviso.

☼ **macOS:** Apple Silicon (M1+). A inferência CorridorKey roda nativamente via MLX (1,5-2x mais rápida que MPS). Geradores de alfa intensivos em GPU (SAM2, GVM, VideoMaMa, MatAnyone2) rodam em MPS, mas são muito mais lentos. No Mac, é recomendado importar mattes alfa prontos.

**O que o instalador faz:**

☼ Verifica [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (compilador C++ exigido pelo OpenEXR) e oferece instalação automática se estiver ausente

☼ Instala [uv](https://docs.astral.sh/uv/) e provisiona Python 3.11 gerenciado para o caminho do instalador

☼ Cria um ambiente virtual `.venv` na pasta do projeto

☼ Instala o backend PyTorch correto para sua plataforma/GPU e verifica o runtime torch resultante antes de informar sucesso

☼ Baixa e instala [FFmpeg](https://ffmpeg.org/) localmente se não for encontrado no PATH (usado para importação de vídeo)

☼ Baixa o checkpoint do modelo CorridorKey (383 MB, obrigatório)

☼ Instala opcionalmente o suporte a rastreamento SAM2 e pré-baixa o checkpoint Base+ padrão (324 MB)

☼ Baixa opcionalmente os geradores de alpha hint BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB) e VideoMaMa (~37 GB)

☼ Cria um atalho na área de trabalho (opcional)

**Atualização:**

☼ **Usuários do instalador Windows Desktop App:** o app verifica atualizações automaticamente. Quando uma nova versão estiver disponível, clique no botão de atualização no app. Ele baixa um patch leve e reinicia.

☼ **Usuários do macOS Desktop App:** O app verifica atualizações automaticamente. Quando uma nova versão estiver disponível, clique no botão de atualização no app. Ele baixa um patch leve e reinicia.

☼ **Usuários CLI:** clique duas vezes em `3-update.bat` (Windows) ou execute `./3-update.sh` (macOS/Linux). Isso baixa o código mais recente via git ou baixa um ZIP se git não estiver disponível.

> **Nota:** O ZIP de atualização no GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) é apenas para usuários do Windows Desktop App Installer. Ele aplica patch a uma instalação existente. Usuários CLI devem continuar usando `3-update.bat` / `3-update.sh`.

### Instalação alternativa: Docker

Para usuários Linux ou configurações remotas/cloud, o EZ-CorridorKey pode rodar dentro do Docker com acesso pelo navegador via noVNC. Consulte [docker/README.md](docker/README.md) para instruções de configuração. A instalação nativa acima é recomendada para Windows e macOS.

## Desinstalação

### Instalador do app desktop

**Windows:**
☼ Abra Settings > Apps > Installed Apps

☼ Encontre **EZ-CorridorKey** e clique em Uninstall

☼ Isso remove o aplicativo e o atalho do menu Start

☼ Seus projetos e modelos baixados ficam em `%APPDATA%\EZ-CorridorKey\`. Exclua essa pasta para remover todos os dados de usuário.

**macOS:**
☼ Arraste `/Applications/EZ-CorridorKey.app` para a Lixeira

☼ Seus projetos, preferências e modelos baixados ficam em `~/Library/Application Support/EZ-CorridorKey/`. Exclua essa pasta para remover todos os dados de usuário.

### Instalação CLI (git clone)

☼ Exclua a pasta do repositório clonado (por exemplo, `EZ-CorridorKey/`). Isso inclui o ambiente virtual `.venv`, os modelos baixados e todos os dados do projeto dentro de `Projects/`.

☼ Se você criou um atalho na área de trabalho durante a instalação, exclua-o manualmente.

☼ A instalação CLI não modifica arquivos no nível do sistema. Não há mais nada para limpar.

### Cache de modelos Hugging Face

Alguns modelos opcionais (BiRefNet, SAM2) são baixados pelo Hugging Face Hub e armazenados em cache fora da pasta do projeto. Para recuperar esse espaço em disco:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Esse cache é compartilhado por todos os aplicativos que usam Hugging Face. Se você usa outras ferramentas de IA, exclua apenas as pastas dos modelos específicos (por exemplo, `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`) em vez do diretório `hub/` inteiro.

## Início rápido

### 1. Importar

Solte um arquivo de vídeo na tela de boas-vindas (ou File > Import Clips > Import Video). O vídeo é extraído automaticamente para uma sequência de imagens de alta qualidade:

1. **FFmpeg extrai** cada quadro como **EXR half-float** (compressão ZIP16)
2. **Passo de recompressão** converte ZIP16 -> **DWAB** (compressão com perdas padrão VFX, ~4x menor)

Essa abordagem em dois passos preserva toda a precisão floating-point do decodificador de vídeo: sem quantização de 8 bits, sem banding. Mesmo vídeos de origem em 8 bits se beneficiam, porque a conversão interna YUV->RGB do FFmpeg permanece em float, evitando erros de arredondamento que se acumulam em pipelines inteiros. Os quadros EXR comprimidos em DWAB costumam ter tamanho parecido com PNG, mantendo o alcance dinâmico half-float de 16 bits.

A recompressão roda em um processo separado, então a interface continua totalmente responsiva durante a extração.

### 2. Gerar alpha hint

Seu clipe começa no estado **RAW** (badge cinza). Você precisa de um alpha hint antes de executar a inferência.

**Opção A - Chroma Key (manual, sem modelo de IA):**
Clique em **CHROMA KEY** no painel de parâmetros. Keyer por diferença de cor para filmagens em tela verde ou azul. Pressione **E** para ativar o conta-gotas e arraste pela tela para amostrar um intervalo de cores. Ajuste Key Strength, Clip Black e Clip White para refinar o matte. Clique em **GENERATE** para produzir o alpha hint.

**Opção B - Geradores de alfa one-click:**
☼ **GVM Auto** - Clique em **GVM AUTO** no painel de parâmetros. Funciona muito bem para a maioria das filmagens em tela verde com pessoas.

☼ **BiRefNet** (recomendado) - Clique em **BIREFNET** no painel de parâmetros. É rápido, preciso e funciona bem em muitos tipos de filmagem.


**Opção C - MatAnyone2 / VideoMaMa:**
Esses modelos precisam de uma dica de máscara. Há duas formas de fornecer uma:

*C1. Pintar + rastrear (do zero):*
1. Pressione **1** para ativar o modo primeiro plano (verde)
2. Pinte sobre o sujeito em alguns quadros-chave
3. Pressione **2** para mudar para o modo fundo (vermelho)
4. Pinte sobre áreas do fundo
5. Clique em **TRACK MASK** para gerar uma trilha de máscara SAM2 densa
6. Clique em **MATANYONE2** ou **VIDEOMAMA** no painel de parâmetros

*C2. Importar máscara (traga a sua):*
1. Clique no botão **+** ao lado de **VIDEOMAMA** para importar uma sequência de máscaras ou um vídeo de máscara pronto
2. Clique em **MATANYONE2** ou **VIDEOMAMA** para executar o modelo

**Opção D - Importar alfa (traga o seu):**
Se você já tem mattes alfa de outra ferramenta (Rotobrush, Silhouette, Resolve, Nuke etc.), clique em **IMPORT ALPHA** no painel de parâmetros e escolha uma pasta de imagens ou um arquivo de vídeo matte.

☼ Formatos de imagem suportados: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Caminho de vídeo alfa suportado: arquivos de vídeo padrão aceitos pelo importador normal de clipes (por exemplo, **MOV** ou **MP4**)

☼ Imagens e vídeos matte visíveis devem estar em **tons de cinza** (branco = primeiro plano, preto = fundo)

☼ A contagem de quadros deve corresponder à sua sequência de entrada

☼ Imagens estáticas que não sejam PNG são convertidas automaticamente para PNG em tons de cinza na importação

☼ Vídeos alfa importados são decodificados em quadros `AlphaHint/*.png` em tons de cinza, então seguem o mesmo caminho downstream dos hints de sequência de imagens

☼ Os arquivos importados são copiados para a pasta `AlphaHint/` do clipe e o clipe avança para o estado **READY**

Você pode reimportar a qualquer momento. Se o clipe já tiver alpha hints, será perguntado se você quer sobrescrevê-los.

### 3. Executar inferência

Seu clipe agora está **READY** (badge amarelo). Ajuste os parâmetros conforme necessário e clique em **RUN INFERENCE** (ou **Ctrl+R**).

### Pipeline em lote (sem supervisão)

Selecione vários clipes na bandeja I/O (Ctrl+clique ou Shift+clique) e clique em **RUN PIPELINE** na barra de status. O sistema automaticamente:

1. Detecta o estado e as pinceladas de cada clipe
2. Executa **TRACK MASK** e depois **VideoMaMa** para clipes pintados
3. Executa **GVM Auto** para clipes RAW sem pintura
4. Encadeia a **inferência** depois que cada geração de alfa termina

Toda a pipeline pode ser cancelada (Esc) e usa checkpoints. Se for interrompida, reiniciar continua de onde cada clipe parou.

### 4. Revisar

Alterne entre os modos de visualização para inspecionar os resultados:

☼ **COMP** - Key sobre fundo quadriculado

☼ **FG** - Verificar fringing verde

☼ **MATTE** - Inspecionar a qualidade do alfa

☼ **PROCESSED** - RGBA de produção

As saídas são gravadas nos subdiretórios `Output/` do projeto durante a inferência (configurável, veja [Pasta de saída personalizada](#pasta-de-saída-personalizada)).

## Requisitos de hardware

### Windows / Linux (NVIDIA CUDA)

|          | Mínimo        | Recomendado        | Confortável                   |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 GB          | 12 GB              | 16 GB+                        |

#### Modos de VRAM

| Modo                  | Uso de VRAM | Velocidade 4K | Como funciona                           |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Refinador em quadro completo, torch.compile completo |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | Refinador em tiles 512x512, compile seletivo |

O modo é detectado automaticamente pela VRAM disponível. Sobrescreva com `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

> Aplica-se à v2.0.0 e instalações a partir do código-fonte. Lançamentos de patch como o 2.1.2 são entregues como atualização automática dentro do app, não como um novo instalador para Mac.

|             | Mínimo        | Recomendado                                      |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (detectado automaticamente se corridorkey-mlx estiver instalado) |

A inferência CorridorKey seleciona automaticamente o backend mais rápido disponível: MLX (1,5-2x mais rápido) quando `corridorkey-mlx` e um checkpoint `.safetensors` estão presentes, caso contrário PyTorch MPS. Sobrescreva com `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Geradores de alfa (SAM2, GVM, VideoMaMa, MatAnyone2) sempre rodam em PyTorch MPS. Não existem ports MLX para esses modelos. Para a melhor experiência no Mac, importe mattes alfa prontos do After Effects, DaVinci Resolve ou Nuke.

> **Problema conhecido:** A saída FG do MLX pode produzir resultados blocados ou com artefatos. Desative a saída FG e use as saídas Matte + Processed. Usuários CUDA não são afetados.

### Atualizações assinadas

A partir da v1.10.0, todo lançamento no GitHub inclui um manifest assinado (`manifest.json` + `manifest.json.sig`). Quando o atualizador dentro do app baixa uma nova versão, ele verifica a assinatura Ed25519 e o hash SHA-256 antes de aplicar a atualização. Se qualquer verificação falhar, a atualização é rejeitada e o usuário recebe um aviso.

Isso significa que, mesmo que a conta do GitHub fosse comprometida, o invasor não poderia enviar uma atualização maliciosa aos usuários instalados sem também possuir a chave de assinatura offline.

O código de verificação é open source: [`backend/update_verify.py`](backend/update_verify.py).

### Checksums

Cada lançamento inclui um arquivo `SHA256SUMS.txt` listando o hash SHA-256 de cada artefato de lançamento. Use-o para confirmar que seu download é genuíno e não está corrompido. Coloque o `SHA256SUMS.txt` na mesma pasta do arquivo baixado e siga os passos para o seu sistema.

**Windows (PowerShell):**

```powershell
# Na pasta de downloads (Shift + clique direito > "Abrir janela do PowerShell aqui")
Get-FileHash .\EZ-CorridorKey-2.1.0-Windows-x64-Setup.exe -Algorithm SHA256
# Abra o SHA256SUMS.txt, encontre a linha que termina com esse nome de arquivo e compare os hashes.
# As duas strings devem corresponder exatamente. Se forem diferentes, não execute o arquivo; baixe novamente.
```

**macOS (Terminal):**

```bash
cd ~/Downloads   # onde você salvou os arquivos
shasum -a 256 -c SHA256SUMS.txt --ignore-missing
# Esperado: EZ-CorridorKey-2.0.0-macOS-arm64.dmg: OK
# Se aparecer FAILED, o arquivo está corrompido ou adulterado; baixe novamente.
```

**Linux:**

```bash
cd ~/Downloads
sha256sum -c SHA256SUMS.txt --ignore-missing
# Esperado: OK ao lado de cada arquivo baixado.
```

## Localização

O EZ-CorridorKey aceita tradução para qualquer idioma. Todas as strings da UI são extraídas para arquivos Qt `.ts` padrão que tradutores podem editar com [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (gratuito) ou qualquer editor de texto.

**Como adicionar seu idioma:**

1. Copie `ui/translations/corridorkey_en.ts` para o código do seu idioma (por exemplo, `corridorkey_fr.ts` para francês)
2. Abra o novo arquivo no Qt Linguist e preencha as traduções
3. Compile: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Envie um pull request com os arquivos `.ts` e `.qm`

Instruções completas com exemplos estão em [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Idiomas atuais:** inglês mais alemão, espanhol, francês, hindi, indonésio, italiano, japonês, coreano, polonês, português (Brasil), russo, turco, ucraniano, vietnamita, chinês (simplificado) e chinês (tradicional, Taiwan). Escolha o seu em Edit > Preferences. Quer ver seu idioma aqui ou melhorar uma tradução existente? PRs são bem-vindos.
