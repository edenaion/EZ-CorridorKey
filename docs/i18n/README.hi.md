## इंस्टॉलेशन

### डेस्कटॉप ऐप इंस्टॉलर (अनुशंसित)

**Python, git या command line से नहीं निपटना चाहते?** पूरा Windows installer, portable exe और macOS `.pkg` उपलब्ध हैं। ये पूरी तरह optional और free हैं। Donations active development को सपोर्ट करते हैं।

[![Gumroad पर डाउनलोड करें](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

Installer में सब कुछ शामिल है: Python runtime, AI models, GPU libraries। सेटअप की ज़रूरत नहीं। बस इंस्टॉल करें और चलाएं।

### टर्मिनल (CLI) इंस्टॉल (Windows / macOS / Linux)

1. इस repository को clone या download करें।
2. One-click path अपने आप managed Python 3.11 तैयार करता है और इस्तेमाल करता है, इसलिए सिर्फ़ `1-install` चलाने के लिए Python पहले से install करने की ज़रूरत नहीं।
3. अपने platform के लिए installer चलाएं:
   ☼ **Windows:** `1-install.bat` पर double-click करें

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. Installer सब कुछ संभालता है: managed Python, virtual environment, dependencies (उपलब्ध होने पर आपके GPU के लिए सही PyTorch backend सहित), verification और model downloads।
5. Launch करने के लिए: `2-start.bat` (Windows) पर double-click करें या `./2-start.sh` (macOS/Linux) चलाएं।

**Prerequisites:**

☼ One-click installer के लिए: preinstalled Python की ज़रूरत नहीं

☼ Manual installs के लिए: [Python 3.10–3.13](https://python.org) (3.14 अभी समर्थित नहीं है)

☼ **Windows/Linux:** CUDA support वाला NVIDIA GPU (8 GB+ VRAM recommended)। Driver current रखें। Installer torch runtime verify करता है और गलत backend पर चुपचाप छोड़ने के बजाय diagnostics के साथ रुकता है।

☼ **macOS:** Apple Silicon (M1+)। CorridorKey inference MLX के ज़रिए native चलता है (MPS से 1.5-2x तेज़)। GPU-intensive alpha generators (SAM2, GVM, VideoMaMa, MatAnyone2) MPS पर चलते हैं, पर काफ़ी धीमे हैं। Mac पर pre-made alpha mattes इम्पोर्ट करना recommended है।

**Installer क्या करता है:**

☼ [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) की जांच करता है (OpenEXR के लिए C++ compiler चाहिए) और missing होने पर auto-install का विकल्प देता है

☼ [uv](https://docs.astral.sh/uv/) install करता है और installer path के लिए managed Python 3.11 तैयार करता है

☼ Project folder में `.venv` virtual environment बनाता है

☼ आपके platform/GPU के लिए सही PyTorch backend install करता है और success बताने से पहले resulting torch runtime verify करता है

☼ PATH पर न मिलने पर [FFmpeg](https://ffmpeg.org/) को local install करता है (video import के लिए इस्तेमाल होता है)

☼ CorridorKey model checkpoint डाउनलोड करता है (383 MB, required)

☼ Optional SAM2 tracking support install करता है और default Base+ checkpoint pre-download करता है (324 MB)

☼ Optional BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB) और VideoMaMa (~37 GB) alpha hint generators डाउनलोड करता है

☼ Desktop shortcut बनाता है (optional)

**अपडेट करना:**

☼ **Windows Desktop App Installer users:** ऐप updates अपने आप check करती है। नई version available होने पर ऐप में update button क्लिक करें। यह lightweight patch डाउनलोड करती है और relaunch करती है।

☼ **macOS Desktop App users:** ऐप updates अपने आप check करती है। नई version available होने पर ऐप में update button क्लिक करें। यह lightweight patch डाउनलोड करती है और relaunch करती है।

☼ **CLI users:** `3-update.bat` (Windows) पर double-click करें या `./3-update.sh` (macOS/Linux) चलाएं। यह git के ज़रिए latest code pull करता है, या git available न हो तो ZIP download करता है।

> **नोट:** GitHub Releases पर update ZIP (`EZ-CorridorKey-windows-x64.zip`) सिर्फ़ Windows Desktop App Installer users के लिए है। यह existing installation को patch करता है। CLI users को `3-update.bat` / `3-update.sh` इस्तेमाल करते रहना चाहिए।

### वैकल्पिक इंस्टॉलेशन: Docker

Linux users या remote/cloud setups के लिए EZ-CorridorKey, noVNC के ज़रिए browser-based access के साथ Docker के अंदर चल सकता है। Setup instructions के लिए [docker/README.md](docker/README.md) देखें। ऊपर दिया native install Windows और macOS के लिए recommended है।

### CLI इंस्टॉल (git clone)

☼ Install के दौरान desktop shortcut बनाया था, तो उसे manually delete करें।

### Hugging Face मॉडल कैश

कुछ optional models (BiRefNet, SAM2) Hugging Face Hub के ज़रिए download होते हैं और project folder के बाहर cache होते हैं। वह disk space वापस पाने के लिए:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

यह cache Hugging Face इस्तेमाल करने वाली सभी applications के बीच shared है। अगर आप दूसरे AI tools इस्तेमाल करते हैं, तो पूरे `hub/` directory के बजाय specific model folders (जैसे `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`) ही delete करें।

## क्विक स्टार्ट

### 1. इम्पोर्ट

Welcome screen पर video file drop करें (या File > Import Clips > Import Video)। Video अपने आप high-quality image sequence में extract होता है:

1. **FFmpeg extracts** each frame as **EXR half-float** (ZIP16 compression)
2. **Recompression pass** converts ZIP16 → **DWAB** (VFX-standard lossy compression, ~4× smaller)

यह two-pass approach video decoder से full floating-point precision बचाकर रखती है: 8-bit quantization नहीं, banding नहीं। 8-bit source video को भी फ़ायदा होता है, क्योंकि FFmpeg का internal YUV→RGB conversion float में रहता है और integer pipelines में जमा होने वाली rounding errors से बचता है। DWAB-compressed EXR frames आम तौर पर PNG जैसे size के होते हैं, लेकिन 16-bit half-float dynamic range रखते हैं।

Recompression अलग process में चलता है, इसलिए extraction के दौरान UI पूरी तरह responsive रहता है।

### 2. अल्फा हिंट जनरेट करें

आपकी क्लिप **RAW** state (gray badge) में शुरू होती है। Inference चलाने से पहले आपको alpha hint चाहिए।

**Option A - Chroma Key (manual, कोई AI model नहीं):**
Parameter panel में **CHROMA KEY** क्लिक करें। यह green screen या blue screen footage के लिए color-difference keyer है। Eyedropper activate करने के लिए **E** दबाएं, फिर colors की range sample करने के लिए screen पर drag करें। Matte refine करने के लिए Key Strength, Clip Black और Clip White adjust करें। Alpha hint बनाने के लिए **GENERATE** क्लिक करें।

**Option B - One-click alpha generators:**
☼ **GVM Auto** - parameter panel में **GVM AUTO** क्लिक करें। लोगों वाले ज़्यादातर green screen footage के लिए अच्छा काम करता है।

☼ **BiRefNet** (recommended) - parameter panel में **BIREFNET** क्लिक करें। तेज़, accurate और footage की wide range पर अच्छा काम करता है।


**Option C - MatAnyone2 / VideoMaMa:**
इन models को mask hint चाहिए। इसे देने के दो तरीके हैं:

*C1. Paint + Track (scratch से):*
1. Foreground mode (green) activate करने के लिए **1** दबाएं
2. कुछ key frames पर subject के ऊपर paint करें
3. Background mode (red) में switch करने के लिए **2** दबाएं
4. Background areas पर paint करें
5. Dense SAM2 mask track generate करने के लिए **TRACK MASK** क्लिक करें
6. Parameter panel में **MATANYONE2** या **VIDEOMAMA** क्लिक करें

*C2. Mask import (अपना लाएं):*
1. Pre-made mask sequence या video import करने के लिए **VIDEOMAMA** के पास **+** button क्लिक करें
2. Model चलाने के लिए **MATANYONE2** या **VIDEOMAMA** क्लिक करें

**Option D - Import Alpha (अपना लाएं):**
अगर आपके पास दूसरे tool (Rotobrush, Silhouette, Resolve, Nuke आदि) से alpha mattes हैं, तो parameter panel में **IMPORT ALPHA** क्लिक करें और image folder या matte video file चुनें।

☼ Supported image formats: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Supported alpha-video path: normal clip importer द्वारा accepted standard video files (जैसे **MOV** या **MP4**)

☼ Images और visible matte videos **grayscale** होने चाहिए (white = foreground, black = background)

☼ Frame count आपकी input sequence से match होना चाहिए

☼ Non-PNG stills import पर अपने आप grayscale PNG में convert होते हैं

☼ Imported alpha videos grayscale `AlphaHint/*.png` frames में decode होते हैं, ताकि वे image-sequence hints जैसा ही downstream path follow करें

☼ Imported files clip के `AlphaHint/` folder में copy होते हैं और clip **READY** state में आगे बढ़ती है

आप कभी भी re-import कर सकते हैं। अगर clip में पहले से alpha hints हैं, तो आपसे पूछा जाएगा कि क्या आप उन्हें overwrite करना चाहते हैं।

### 3. इन्फरेंस चलाएं

आपकी clip अब **READY** (yellow badge) है। ज़रूरत के अनुसार parameters adjust करें, फिर **RUN INFERENCE** क्लिक करें (या **Ctrl+R**)।

### बैच पाइपलाइन (बिना निगरानी)

I/O tray में multiple clips select करें (Ctrl+click या Shift+click), फिर status bar में **RUN PIPELINE** क्लिक करें। System automatically:

1. हर clip की state और paint strokes detect करता है
2. Painted clips के लिए **TRACK MASK** और फिर **VideoMaMa** चलाता है
3. Unpainted RAW clips के लिए **GVM Auto** चलाता है
4. हर alpha generation पूरा होने के बाद **inference** chain करता है

पूरा pipeline cancellable (Esc) और checkpointable है। अगर बीच में interrupt हो, तो restart करने पर हर clip वहीं से आगे बढ़ती है जहां छोड़ा था।

### 4. रिव्यू

Results inspect करने के लिए view modes के बीच switch करें:

☼ **COMP** - checkerboard पर key

☼ **FG** - green fringing check करें

☼ **MATTE** - alpha quality inspect करें

☼ **PROCESSED** - production RGBA

Inference के दौरान outputs project के `Output/` subdirectories में लिखे जाते हैं (configurable, [Custom Output Directory](#कसटम-आउटपट-डयरकटर) देखें)।

## लोकलाइज़ेशन

EZ-CorridorKey किसी भी भाषा में translation support करता है। सभी UI strings standard Qt `.ts` files में extracted हैं, जिन्हें translators [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (free) या किसी भी text editor से edit कर सकते हैं।

**अपनी भाषा कैसे जोड़ें:**

1. `ui/translations/corridorkey_en.ts` को अपने language code पर copy करें (जैसे French के लिए `corridorkey_fr.ts`)
2. नई file Qt Linguist में open करें और translations भरें
3. Compile करें: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. `.ts` और `.qm` दोनों files के साथ pull request submit करें

Examples सहित full instructions [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md) में हैं।

**वर्तमान भाषाएं:** English के अलावा German, Spanish, French, Hindi, Indonesian, Italian, Japanese, Korean, Polish, Portuguese (Brazil), Russian, Turkish, Ukrainian, Vietnamese, Chinese (Simplified), और Chinese (Traditional, Taiwan)। Edit > Preferences में अपनी भाषा चुनें। अपनी भाषा यहां देखना चाहते हैं या किसी translation को बेहतर बनाना चाहते हैं? PRs welcome।
