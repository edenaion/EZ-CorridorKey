## Instalasi

**Lebih suka video?** Tonton [tutorial langkah demi langkah](https://www.youtube.com/watch?v=ezg_1oQzz7w) dari pembuat aplikasi:

<a href="https://www.youtube.com/watch?v=ezg_1oQzz7w"><img src="https://img.youtube.com/vi/ezg_1oQzz7w/maxresdefault.jpg" alt="EZ-CorridorKey video tutorial" width="640"></a>

### Installer aplikasi desktop (direkomendasikan)

**Tidak ingin berurusan dengan Python, git, atau command line?** Installer Windows lengkap, exe portabel, dan `.pkg` macOS tersedia. Sepenuhnya opsional dan gratis, donasi membantu mendukung pengembangan aktif.

[![Unduh di Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

Installer menyertakan semuanya: runtime Python, model AI, library GPU. Tidak perlu penyiapan. Cukup instal dan jalankan.

### Instalasi terminal (CLI) (Windows / macOS / Linux)

1. Clone atau unduh repositori ini.
2. Jalur one-click menyediakan dan memakai Python 3.11 terkelola secara otomatis, jadi Anda tidak perlu memasang Python lebih dulu hanya untuk memakai `1-install`.
3. Jalankan installer untuk platform Anda:
   ☼ **Windows:** Klik dua kali `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. Installer menangani semuanya: Python terkelola, virtual environment, dependency (termasuk backend PyTorch yang tepat untuk GPU Anda jika tersedia), verifikasi, dan unduhan model.
5. Untuk meluncurkan: klik dua kali `2-start.bat` (Windows) atau `./2-start.sh` (macOS/Linux).

**Prasyarat:**

☼ Untuk installer one-click: Python prainstal tidak diperlukan

☼ Untuk instalasi manual: [Python 3.10–3.13](https://python.org) (3.14 belum didukung)

☼ **Windows/Linux:** GPU NVIDIA dengan dukungan CUDA (VRAM 8 GB+ direkomendasikan). Pastikan driver tetap baru. Installer memverifikasi runtime torch dan akan berhenti dengan diagnostik, bukan diam-diam meninggalkan Anda pada backend yang salah.

☼ **macOS:** Apple Silicon (M1+). Inferensi CorridorKey berjalan native melalui MLX (1,5-2x lebih cepat daripada MPS). Generator alpha yang intensif GPU (SAM2, GVM, VideoMaMa, MatAnyone2) berjalan di MPS tetapi jauh lebih lambat. Mengimpor matte alpha siap pakai direkomendasikan di Mac.

**Yang dilakukan installer:**

☼ Memeriksa [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (compiler C++ yang dibutuhkan OpenEXR), dan menawarkan auto-install jika belum ada

☼ Menginstal [uv](https://docs.astral.sh/uv/) dan menyediakan Python 3.11 terkelola untuk jalur installer

☼ Membuat virtual environment `.venv` di folder proyek

☼ Menginstal backend PyTorch yang tepat untuk platform/GPU Anda dan memverifikasi runtime torch yang dihasilkan sebelum melaporkan sukses

☼ Mengunduh dan menginstal [FFmpeg](https://ffmpeg.org/) secara lokal jika tidak ditemukan di PATH (dipakai untuk impor video)

☼ Mengunduh checkpoint model CorridorKey (383 MB, wajib)

☼ Opsional: menginstal dukungan tracking SAM2 dan mengunduh lebih dulu checkpoint Base+ default (324 MB)

☼ Opsional: mengunduh generator alpha hint BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB), dan VideoMaMa (~37 GB)

☼ Membuat pintasan desktop (opsional)

**Update:**

☼ **Pengguna installer aplikasi desktop Windows:** Aplikasi memeriksa update secara otomatis. Saat versi baru tersedia, klik tombol update di aplikasi. Aplikasi mengunduh patch ringan dan meluncur ulang.

☼ **Pengguna macOS Desktop App:** Aplikasi memeriksa update secara otomatis. Saat versi baru tersedia, klik tombol update di aplikasi. Aplikasi mengunduh patch ringan dan meluncur ulang.

☼ **Pengguna CLI:** Klik dua kali `3-update.bat` (Windows) atau jalankan `./3-update.sh` (macOS/Linux). Ini mengambil kode terbaru melalui git, atau mengunduh ZIP jika git tidak tersedia.

> **Catatan:** ZIP update di GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) hanya untuk pengguna installer aplikasi desktop Windows. ZIP ini menambal instalasi yang sudah ada. Pengguna CLI harus tetap memakai `3-update.bat` / `3-update.sh`.

### Instalasi alternatif: Docker

Untuk pengguna Linux atau setup remote/cloud, EZ-CorridorKey dapat berjalan di dalam Docker dengan akses berbasis browser melalui noVNC. Lihat [docker/README.md](docker/README.md) untuk instruksi penyiapan. Instalasi native di atas direkomendasikan untuk Windows dan macOS.

## Menghapus instalasi

### Installer aplikasi desktop

**Windows:**
☼ Buka Settings > Apps > Installed Apps

☼ Cari **EZ-CorridorKey** dan klik Uninstall

☼ Ini menghapus aplikasi dan pintasan Start Menu

☼ Proyek dan model yang diunduh disimpan di `%APPDATA%\EZ-CorridorKey\`. Hapus folder itu untuk menghapus semua data pengguna.

**macOS:**
☼ Seret `/Applications/EZ-CorridorKey.app` ke Trash

☼ Proyek, preferensi, dan model yang diunduh disimpan di `~/Library/Application Support/EZ-CorridorKey/`. Hapus folder itu untuk menghapus semua data pengguna.

### Instalasi CLI (git clone)

☼ Hapus folder repositori hasil clone (mis. `EZ-CorridorKey/`). Ini mencakup virtual environment `.venv`, model yang diunduh, dan semua data proyek di dalam `Projects/`.

☼ Jika Anda membuat pintasan desktop saat instalasi, hapus secara manual.

☼ Tidak ada file tingkat sistem yang diubah oleh instalasi CLI. Tidak ada lagi yang perlu dibersihkan.

### Cache model Hugging Face

Beberapa model opsional (BiRefNet, SAM2) diunduh melalui Hugging Face Hub dan di-cache di luar folder proyek. Untuk mengambil kembali ruang disk itu:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Cache ini dipakai bersama oleh semua aplikasi yang menggunakan Hugging Face. Jika Anda memakai tool AI lain, hapus hanya folder model tertentu (mis. `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`), bukan seluruh direktori `hub/`.

## Mulai cepat

### 1. Impor

Jatuhkan file video ke layar pembuka (atau File > Impor Klip > Impor Video). Video otomatis diekstrak menjadi sekuens gambar berkualitas tinggi:

1. **FFmpeg extracts** setiap frame sebagai **EXR half-float** (kompresi ZIP16)
2. **Recompression pass** mengonversi ZIP16 → **DWAB** (kompresi lossy standar VFX, ~4x lebih kecil)

Pendekatan dua pass ini mempertahankan presisi floating-point penuh dari decoder video: tidak ada kuantisasi 8-bit, tidak ada banding. Bahkan video sumber 8-bit mendapat manfaat karena konversi YUV→RGB internal FFmpeg tetap dalam float, menghindari error pembulatan yang menumpuk di pipeline integer. Frame EXR terkompresi DWAB biasanya berukuran sebanding dengan PNG sambil mempertahankan dynamic range half-float 16-bit.

Rekompresi berjalan dalam proses terpisah sehingga UI tetap responsif penuh selama ekstraksi.

### 2. Buat alpha hint

Klip Anda mulai dalam status **RAW** (badge abu-abu). Anda memerlukan alpha hint sebelum menjalankan inferensi.

**Opsi A - Chroma Key (manual, tanpa model AI):**
Klik **CHROMA KEY** di panel parameter. Keyer berbasis perbedaan warna untuk footage layar hijau atau layar biru. Tekan **E** untuk mengaktifkan eyedropper, lalu seret melintasi layar untuk mengambil sampel rentang warna. Sesuaikan Key Strength, Clip Black, dan Clip White untuk menyempurnakan matte. Klik **GENERATE** untuk membuat alpha hint.

**Opsi B - Generator alpha sekali klik:**
☼ **GVM Auto** - Klik **GVM AUTO** di panel parameter. Sangat baik untuk sebagian besar footage layar hijau dengan orang.

☼ **BiRefNet** (direkomendasikan) - Klik **BIREFNET** di panel parameter. Cepat, akurat, dan bekerja baik pada banyak jenis footage.


**Opsi C - MatAnyone2 / VideoMaMa:**
Model ini membutuhkan mask hint. Ada dua cara untuk memberikannya:

*C1. Paint + Track (dari nol):*
1. Tekan **1** untuk mengaktifkan mode latar depan (hijau)
2. Sapukan kuas di atas subjek pada beberapa key frame
3. Tekan **2** untuk beralih ke mode latar belakang (merah)
4. Sapukan kuas di atas area latar belakang
5. Klik **TRACK MASK** untuk membuat track mask SAM2 yang padat
6. Klik **MATANYONE2** atau **VIDEOMAMA** di panel parameter

*C2. Impor mask (bawa sendiri):*
1. Klik tombol **+** di sebelah **VIDEOMAMA** untuk mengimpor sekuens mask siap pakai atau video
2. Klik **MATANYONE2** atau **VIDEOMAMA** untuk menjalankan model

**Opsi D - Impor Alpha (bawa sendiri):**
Jika Anda sudah memiliki matte alpha dari tool lain (Rotobrush, Silhouette, Resolve, Nuke, dll.), klik **IMPORT ALPHA** di panel parameter dan pilih folder gambar atau file video matte.

☼ Format gambar yang didukung: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Jalur video alpha yang didukung: file video standar yang diterima oleh pengimpor klip normal (misalnya **MOV** atau **MP4**)

☼ Gambar dan video matte yang terlihat harus **grayscale** (putih = latar depan, hitam = latar belakang)

☼ Jumlah frame harus cocok dengan sekuens input Anda

☼ Still non-PNG otomatis dikonversi menjadi PNG grayscale saat impor

☼ Video alpha yang diimpor didekode menjadi frame grayscale `AlphaHint/*.png` sehingga mengikuti jalur downstream yang sama dengan hint sekuens gambar

☼ File yang diimpor disalin ke folder `AlphaHint/` milik klip dan klip maju ke status **READY**

Anda dapat mengimpor ulang kapan saja. Jika klip sudah memiliki alpha hint, Anda akan diminta memilih apakah akan menimpanya.

### 3. Jalankan inferensi

Klip Anda sekarang **READY** (badge kuning). Sesuaikan parameter sesuai kebutuhan, lalu klik **RUN INFERENCE** (atau **Ctrl+R**).

### Batch pipeline (tanpa pengawasan)

Pilih beberapa klip di tray I/O (Ctrl+klik atau Shift+klik), lalu klik **RUN PIPELINE** di status bar. Sistem otomatis:

1. Mendeteksi status dan sapuan tiap klip
2. Menjalankan **TRACK MASK** lalu **VideoMaMa** untuk klip yang dilukis
3. Menjalankan **GVM Auto** untuk klip RAW yang tidak dilukis
4. Merangkai **inferensi** setelah setiap pembuatan alpha selesai

Seluruh pipeline dapat dibatalkan (Esc) dan dapat dilanjutkan dari checkpoint. Jika terputus, restart akan melanjutkan dari posisi terakhir tiap klip.

### 4. Tinjau

Beralih antar mode tampilan untuk memeriksa hasil:

☼ **COMP** - Key di atas checkerboard

☼ **FG** - Periksa green fringing

☼ **MATTE** - Periksa kualitas alpha

☼ **PROCESSED** - RGBA produksi

Output ditulis ke subdirektori `Output/` proyek selama inferensi (dapat dikonfigurasi, lihat [Direktori output kustom](#direktori-output-kustom)).

## Scrubber frame

Scrubber di bawah penampil ganda menyediakan:

☼ **Tombol transport:** Frame pertama, mundur satu langkah, putar/jeda, maju satu langkah, frame terakhir

☼ **Playback is CAPPED:** Menekan spacebar akan memutar footage pada laju hardcoded 3FPS. Ini disengaja, filenya besar.

☼ **Bilah cakupan:** Tiga jalur berkode warna yang menunjukkan frame mana yang memiliki sapuan (hijau), alpha hint (putih), dan output inferensi (kuning)

☼ **Marker In/Out:** Tekan **I** / **O** untuk mengatur sub-range pemrosesan. Saat diatur, tombol RUN berubah menjadi "RUN SELECTED" dan playback loop dalam rentang tersebut.

## Persyaratan perangkat keras

### Windows / Linux (NVIDIA CUDA)

|          | Minimum       | Direkomendasikan        | Nyaman                   |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 GB          | 12 GB              | 16 GB+                        |
| **GPU**  | NVIDIA (CUDA) | Ampere+ (RTX 30xx) | Ada/Blackwell (RTX 40xx/50xx) |

#### Mode VRAM

| Mode                  | Penggunaan VRAM | Kecepatan 4K    | Cara kerjanya                             |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Refiner full-frame, torch.compile penuh   |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | Refiner tiled 512×512, compile selektif |

Mode terdeteksi otomatis dari VRAM yang tersedia. Override dengan `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

|             | Minimum       | Direkomendasikan                                      |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (terdeteksi otomatis jika corridorkey-mlx terinstal) |

> Berlaku untuk v2.0.0 dan instalasi sumber. Rilis patch seperti 2.1.3 dikirimkan sebagai auto-update di dalam aplikasi, bukan sebagai installer Mac baru.

Inferensi CorridorKey otomatis memilih backend tercepat yang tersedia: MLX (1,5-2x lebih cepat) saat `corridorkey-mlx` dan checkpoint `.safetensors` ada, jika tidak PyTorch MPS. Override dengan `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Generator alpha (SAM2, GVM, VideoMaMa, MatAnyone2) selalu berjalan di PyTorch MPS. Port MLX tidak ada untuk model-model ini. Untuk pengalaman Mac terbaik, impor matte alpha siap pakai dari After Effects, DaVinci Resolve, atau Nuke.

> **Masalah yang diketahui:** Output FG MLX dapat menghasilkan hasil blocky/berartifact. Nonaktifkan output FG dan gunakan output Matte + Processed sebagai gantinya. Pengguna CUDA tidak terpengaruh.

### Update bertanda tangan

Mulai v1.10.0, setiap rilis GitHub mengirim manifest bertanda tangan (`manifest.json` + `manifest.json.sig`). Saat updater di dalam aplikasi mengunduh versi baru, aplikasi memverifikasi tanda tangan Ed25519 dan hash SHA-256 sebelum menerapkan update. Jika salah satu pemeriksaan gagal, update ditolak dan pengguna diperingatkan.

Ini berarti bahkan jika akun GitHub dikompromikan, penyerang tidak dapat mendorong update berbahaya ke pengguna yang sudah memasang aplikasi tanpa juga memiliki signing key offline.

Kode verifikasi bersifat open source: [`backend/update_verify.py`](backend/update_verify.py).

### Checksum

Setiap rilis menyertakan file `SHA256SUMS.txt` yang mencantumkan hash SHA-256 dari setiap artifact rilis. Gunakan file itu untuk mengonfirmasi unduhan Anda asli dan tidak rusak. Letakkan `SHA256SUMS.txt` di folder yang sama dengan file yang Anda unduh, lalu ikuti langkah-langkah untuk sistem Anda.

**Windows (PowerShell):**

```powershell
# Di folder unduhan (Shift + klik kanan > "Open PowerShell window here")
Get-FileHash .\EZ-CorridorKey-2.1.0-Windows-x64-Setup.exe -Algorithm SHA256
# Buka SHA256SUMS.txt, cari baris yang diakhiri nama file tersebut, bandingkan hash-nya.
# Kedua string harus cocok persis. Jika berbeda, jangan jalankan file tersebut, unduh lagi.
```

**macOS (Terminal):**

```bash
cd ~/Downloads   # ke mana pun Anda menyimpan file
shasum -a 256 -c SHA256SUMS.txt --ignore-missing
# Expect: EZ-CorridorKey-2.0.0-macOS-arm64.dmg: OK
# Jika hasilnya FAILED, file rusak atau telah dimanipulasi, unduh lagi.
```

**Linux:**

```bash
cd ~/Downloads
sha256sum -c SHA256SUMS.txt --ignore-missing
# Expect OK di samping setiap file yang Anda unduh.
```

## Lokalisasi

EZ-CorridorKey mendukung terjemahan ke bahasa apa pun. Semua string UI diekstrak ke file Qt `.ts` standar yang dapat diedit penerjemah dengan [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (gratis) atau editor teks apa pun.

**Cara menambahkan bahasa Anda:**

1. Salin `ui/translations/corridorkey_en.ts` ke kode bahasa Anda (mis. `corridorkey_fr.ts` untuk French)
2. Buka file baru di Qt Linguist dan isi terjemahannya
3. Compile: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Kirim pull request dengan file `.ts` dan `.qm`

Instruksi lengkap dengan contoh ada di [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Bahasa saat ini:** English ditambah Jerman, Spanyol, Prancis, Hindi, Indonesia, Italia, Jepang, Korea, Polandia, Portugis (Brasil), Rusia, Turki, Ukraina, Vietnam, Tionghoa (Sederhana), dan Tionghoa (Tradisional, Taiwan). Pilih bahasa Anda di Edit > Preferensi. Ingin melihat bahasa Anda di sini atau menyempurnakan terjemahan? PR diterima.
