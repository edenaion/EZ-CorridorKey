Усуває збій генерації альфа-каналу ШІ з повідомленням «failed to read frame» після імпорту, спричинений дефектними збірками FFmpeg, встановленими попередніми версіями ([#184](https://github.com/edenaion/EZ-CorridorKey/issues/184)); імпорти тепер перевіряються на цілісність з автоматичним повторенням, а також додано необов'язкове звітування про збої, вимкнене за замовчуванням. Дивіться [повний журнал змін](CHANGELOG.md).

Повноцінний графічний інтерфейс для [CorridorKey від Ніко Пюрінгера](https://github.com/nikopueringer/CorridorKey) — AI-хромакею від Corridor Digital, який фізично розмішує передній план від фону, зберігаючи волосся, розмиття руху та напівпрозорість.

Цей інтерфейс замінює робочий процес CLI з перетягуванням файлів на повноцінний застосунок для комп'ютера, зберігаючи 100% зворотну сумісність (`python main.py --cli` досі запускає оригінальний майстер).

## Встановлення

**Зручніше відео?** Перегляньте [покроковий відеоурок](https://www.youtube.com/watch?v=ezg_1oQzz7w) від творця застосунку:

<a href="https://www.youtube.com/watch?v=ezg_1oQzz7w"><img src="https://img.youtube.com/vi/ezg_1oQzz7w/maxresdefault.jpg" alt="EZ-CorridorKey video tutorial" width="640"></a>

### Інсталятор застосунку для комп'ютера (рекомендовано)

**Не хочете мати справу з Python, git або командним рядком?** Доступні повноцінний інсталятор для Windows та портативний exe. Користувачі Mac можуть завантажити `.pkg` для macOS v2.0.0, поки в розробці знаходиться нативний Mac-застосунок. Повністю необов'язково та безплатно; пожертвування підтримують активну розробку.

[![Завантажити на Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

Інсталятор містить усе — середовище Python, AI-моделі, бібліотеки GPU — жодного налаштування не потрібно. Просто встановіть і запустіть.

### Встановлення через термінал (CLI) (Windows / macOS / Linux)

1. Клонуйте або завантажте цей репозиторій.
2. Шлях «в один клік» автоматично підготовлює та використовує керований Python 3.11, тому вам не потрібно попередньо встановлювати Python лише для використання `1-install`.
3. Запустіть інсталятор для вашої платформи:
   ☼ **Windows:** двічі клацніть `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. Інсталятор впорається з усім: керований Python, віртуальне середовище, залежності (включно з правильним бекендом PyTorch для вашого GPU, якщо доступний), перевірка та завантаження моделей.
5. Для запуску: двічі клацніть `2-start.bat` (Windows) або `./2-start.sh` (macOS/Linux).

**Вимоги:**

☼ Для інсталятора «в один клік»: попередньо встановлений Python не потрібен

☼ Для ручного встановлення: [Python 3.10–3.13](https://python.org) (3.14 ще не підтримується)

☼ **Windows/Linux:** GPU NVIDIA з підтримкою CUDA (рекомендовано 8 ГБ+ VRAM). Тримайте драйвер в актуальному стані — інсталятор перевіряє середовище torch і зупиниться з діагностикою замість того, щоб мовчки залишити вас на неправильному бекенді.

☼ **macOS:** Apple Silicon (M1+). Інференція CorridorKey запускається нативно через MLX (у 1,5–2 рази швидше, ніж MPS). Генератори альфа з інтенсивним використанням GPU (SAM2, GVM, VideoMaMa, MatAnyone2) працюють на MPS, але значно повільніше — на Mac рекомендується імпортувати готові альфа-мати.

**Що робить інсталятор:**

☼ Перевіряє наявність [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (компілятор C++, потрібний для OpenEXR) — пропонує автоматичне встановлення, якщо відсутні

☼ Встановлює [uv](https://docs.astral.sh/uv/) та підготовлює керований Python 3.11 для шляху інсталятора

☼ Створює віртуальне середовище `.venv` у папці проекту

☼ Встановлює правильний бекенд PyTorch для вашої платформи/GPU та перевіряє отримане середовище torch перед повідомленням про успіх

☼ Завантажує та встановлює [FFmpeg](https://ffmpeg.org/) локально, якщо не знайдено у PATH (використовується для імпорту відео)

☼ Завантажує контрольну точку моделі CorridorKey (383 МБ, обов'язково)

☼ Опційно встановлює підтримку трекінгу SAM2 і попередньо завантажує стандартну контрольну точку Base+ (324 МБ)

☼ Опційно завантажує генератори альфа BiRefNet (~940 МБ), MatAnyone2 (~135 МБ), GVM (~6 ГБ) та VideoMaMa (~37 ГБ)

☼ Створює ярлик на робочому столі (опційно)

**Оновлення:**

☼ **Користувачі інсталятора Windows Desktop App:** Застосунок автоматично перевіряє наявність оновлень. Коли з'являється нова версія, натисніть кнопку оновлення у застосунку. Він завантажить невеликий патч і перезапуститься.

☼ **Користувачі macOS Desktop App:** Застосунок автоматично перевіряє наявність оновлень. Коли з'являється нова версія, натисніть кнопку оновлення у застосунку. Він завантажить невеликий патч і перезапуститься.

☼ **Користувачі CLI:** двічі клацніть `3-update.bat` (Windows) або виконайте `./3-update.sh` (macOS/Linux). Це завантажить останній код через git або завантажить ZIP-архів, якщо git недоступний.

> **Примітка:** ZIP-архів оновлення на GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) призначений лише для користувачів інсталятора Windows Desktop App. Він патчить наявне встановлення. Користувачі CLI повинні продовжувати використовувати `3-update.bat` / `3-update.sh`.

### Альтернативне встановлення: Docker

Для користувачів Linux або хмарних/віддалених налаштувань EZ-CorridorKey може працювати у Docker з доступом через браузер за допомогою noVNC. Дивіться [docker/README.md](docker/README.md) для інструкцій з налаштування. Нативне встановлення, описане вище, рекомендується для Windows та macOS.

## Видалення

### Інсталятор застосунку для комп'ютера

**Windows:**
☼ Відкрийте Settings > Apps > Installed Apps

☼ Знайдіть **EZ-CorridorKey** та натисніть Uninstall

☼ Це видалить застосунок і ярлик у меню «Пуск»

☼ Ваші проекти та завантажені моделі зберігаються у `%APPDATA%\EZ-CorridorKey\`. Видаліть цю папку, щоб прибрати всі дані користувача.

**macOS:**
☼ Перетягніть `/Applications/EZ-CorridorKey.app` до Смітника

☼ Ваші проекти, параметри та завантажені моделі зберігаються у `~/Library/Application Support/EZ-CorridorKey/`. Видаліть цю папку, щоб прибрати всі дані користувача.

### Встановлення CLI (git clone)

☼ Видаліть клоновану папку репозиторію (наприклад, `EZ-CorridorKey/`). Це включає віртуальне середовище `.venv`, завантажені моделі та всі дані проекту у `Projects/`.

☼ Якщо ви створювали ярлик на робочому столі під час встановлення, видаліть його вручну.

☼ Встановлення CLI не змінює жодних системних файлів. Більше нічого прибирати не потрібно.

### Кеш моделей Hugging Face

Деякі опційні моделі (BiRefNet, SAM2) завантажуються через Hugging Face Hub і кешуються поза папкою проекту. Щоб звільнити місце на диску:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Цей кеш є спільним для всіх застосунків, що використовують Hugging Face. Якщо ви використовуєте інші AI-інструменти, видаляйте лише конкретні папки моделей (наприклад, `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`), а не весь каталог `hub/`.

☼ **Лоток введення/виведення** — горизонтальна смуга мініатюр під переглядачем

## Швидкий старт

### 1. Імпорт

Перетягніть відеофайл на вітальний екран (або File > Import Clips > Import Video). Відео автоматично розпаковується у високоякісну послідовність зображень:

1. **FFmpeg витягує** кожен кадр як **EXR half-float** (стиснення ZIP16)
2. **Прохід перестиснення** перетворює ZIP16 → **DWAB** (стандартне для VFX стиснення з втратами, ~4× менше)

Цей двопрохідний підхід зберігає повну точність з плаваючою комою від відеодекодера — жодного 8-бітного квантування, жодного бендингу. Навіть 8-бітне вихідне відео виграє від цього, оскільки внутрішнє перетворення YUV→RGB у FFmpeg залишається у форматі float, уникаючи помилок округлення, що накопичуються в цілочисельних конвеєрах. Кадри EXR зі стисненням DWAB зазвичай порівнянні за розміром із PNG, але зберігають динамічний діапазон 16-бітного half-float.

Перестиснення виконується в окремому процесі, щоб інтерфейс залишався повністю чуйним під час витягування.

**Супровідні підказки:** файли з «alphahint» або «maskhint» будь-де в назві файлу (без урахування регістру) автоматично пов'язуються з відповідним кліпом як підказки альфа або маски, а не імпортуються як окремі кліпи. Працює як для імпортованих відео, так і для папок з послідовністю кадрів.

### 2. Згенерувати підказку альфа

Ваш кліп починається у стані **RAW** (сіра позначка). Перед запуском інференції потрібна підказка альфа.

**Варіант A — Хромакей (вручну, без AI-моделі):**
Натисніть **CHROMA KEY** на панелі параметрів. Кеїнг на основі різниці кольорів для матеріалів із зеленим або синім екраном. Натисніть **E**, щоб активувати піпетку, а потім перетягніть по екрану для вибірки діапазону кольорів. Налаштуйте Key Strength, Clip Black та Clip White для уточнення мату. Натисніть **GENERATE**, щоб згенерувати підказку альфа.

**Варіант Б — Генератори альфа в один клік:**
☼ **GVM Auto** — натисніть **GVM AUTO** на панелі параметрів. Відмінно підходить для більшості матеріалів із зеленим екраном із людьми.

☼ **BiRefNet** (рекомендовано) — натисніть **BIREFNET** на панелі параметрів. Швидкий, точний та добре працює з широким спектром відеоматеріалів.

☼ **Apple Vision** (macOS 14+) — натисніть **APPLE VISION** на панелі параметрів. Apple's Neural Engine сегментує передній план без малювання чи анотування. Автоматично приховується на інших платформах.

**Варіант В — MatAnyone2 / VideoMaMa:**
Цим моделям потрібна підказка маски. Два способи надати її:

*В1. Малювання + трекінг (з нуля):*
1. Натисніть **1**, щоб активувати режим переднього плану (зелений)
2. Намалюйте об'єкт на кількох ключових кадрах
3. Натисніть **2**, щоб перейти до режиму фону (червоний)
4. Намалюйте ділянки фону
5. Натисніть **TRACK MASK**, щоб згенерувати щільний трек маски SAM2
6. Натисніть **MATANYONE2** або **VIDEOMAMA** на панелі параметрів

*В2. Імпорт маски (власна маска):*
1. Натисніть кнопку **+** поруч із **VIDEOMAMA**, щоб імпортувати готову послідовність масок або відео
2. Натисніть **MATANYONE2** або **VIDEOMAMA**, щоб запустити модель

**Варіант Г — Імпорт альфа (власний матеріал):**
Якщо у вас вже є альфа-мати з іншого інструменту (Rotobrush, Silhouette, Resolve, Nuke тощо), натисніть **IMPORT ALPHA** на панелі параметрів і виберіть папку із зображеннями або відеофайл мату.

☼ Підтримувані формати зображень: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Підтримуваний шлях альфа-відео: стандартні відеофайли, прийняті звичайним імпортером кліпів (наприклад, **MOV** або **MP4**)

☼ Зображення та видимі мат-відео мають бути **в градаціях сірого** (білий = передній план, чорний = фон)

☼ Кількість кадрів повинна збігатися з вашою вхідною послідовністю

☼ Нерухомі зображення не у форматі PNG автоматично перетворюються у PNG в градаціях сірого під час імпорту

☼ Імпортовані відео альфа декодуються у кадри `AlphaHint/*.png` у градаціях сірого, щоб вони йшли тим самим шляхом подальшого оброблення, що й підказки у вигляді послідовності зображень

☼ Імпортовані файли копіюються до папки `AlphaHint/` кліпу, а кліп переходить до стану **READY**

Ви можете повторно імпортувати в будь-який момент — якщо кліп вже має підказки альфа, вас запитають, чи перезаписати їх.

### 3. Запустити інференцію

Ваш кліп тепер **READY** (жовта позначка). Налаштуйте параметри за потреби, а потім натисніть **RUN INFERENCE** (або **Ctrl+R**).

### Пакетний конвеєр (без нагляду)

Два способи запустити пакет:

**З лотка введення/виведення:** виберіть кілька кліпів (Ctrl+клік або Shift+клік), потім натисніть **RUN PIPELINE** на рядку стану.

**З папки:** File > Batch Pipeline відкриває діалог для оброблення цілої папки з кліпами. Виберіть папку, оберіть, яку модель генерації альфа використовувати (GVM, BiRefNet, VideoMaMa, MatAnyone2), і запустіть усе автономно. Перевизначення для кожного кліпу дозволяють змішувати моделі в одному пакеті. Індикатори прогресу та позначки в реальному часі відстежують стан кожного кліпу. Підпапки з послідовностями зображень визначаються та можуть поєднуватися з відео в одному пакеті.

Для кліпів, вибраних у лотку, система автоматично:

1. Визначає стан кожного кліпу та мазки малювання
2. Запускає **TRACK MASK**, а потім **VideoMaMa** для намальованих кліпів
3. Запускає **GVM Auto** для ненамальованих кліпів у стані RAW
4. Виконує **інференцію** після завершення кожної генерації альфа

Весь конвеєр можна скасувати (Esc) і він підтримує контрольні точки — у разі переривання перезапуск продовжує з того місця, де зупинився кожен кліп.

## Вимоги до апаратного забезпечення

### Windows / Linux (NVIDIA CUDA)

|          | Мінімум       | Рекомендовано      | Комфортно                     |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 ГБ          | 12 ГБ              | 16 ГБ+                        |
| **GPU**  | NVIDIA (CUDA) | Ampere+ (RTX 30xx) | Ada/Blackwell (RTX 40xx/50xx) |

#### Режими VRAM

| Режим                 | Використання VRAM | Швидкість 4K | Принцип роботи                           |
| --------------------- | ----------------- | ------------ | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB           | ~1.5 с/кадр  | Повнокадровий refiner, повний torch.compile   |
| **Low-VRAM** (<12 GB) | ~2.5 GB           | ~1.6 с/кадр  | Мозаїчний refiner 512×512, вибіркова компіляція |

Режим визначається автоматично на основі доступного VRAM. Перевизначте за допомогою `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

> Стосується v2.0.0 та встановлень із джерельного коду. Патч-релізи, такі як 2.1.4, постачаються як вбудоване автоматичне оновлення, а не новий Mac-інсталятор.

|              | Мінімум       | Рекомендовано                                    |
| ------------ | ------------- | ------------------------------------------------ |
| **Чіп**      | M1 (8 ГБ)     | M1 Pro+ (16 ГБ+)                                 |
| **Бекенд**   | MPS (PyTorch) | MLX (вбудований, визначається автоматично)       |

Інференція CorridorKey автоматично вибирає найшвидший доступний бекенд: вбудований рушій MLX (приблизно у 2 рази швидше за MPS, працює у float16), якщо доступні контрольні точки `.mlx.safetensors`, або PyTorch MPS. Обидві контрольні точки MLX — для зеленого та синього — завантажуються через майстер налаштування. Перевизначте за допомогою `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Генератори альфа (SAM2, GVM, VideoMaMa, MatAnyone2) завжди працюють на PyTorch MPS — портів MLX для цих моделей не існує. Для найкращого досвіду на Mac імпортуйте готові альфа-мати з After Effects, DaVinci Resolve або Nuke.

## Локалізація

EZ-CorridorKey підтримує переклад будь-якою мовою. Усі рядки інтерфейсу витягнуті у стандартні файли Qt `.ts`, які перекладачі можуть редагувати у [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (безплатно) або будь-якому текстовому редакторі.

**Як додати свою мову:**

1. Скопіюйте `ui/translations/corridorkey_en.ts` до файлу з кодом вашої мови (наприклад, `corridorkey_fr.ts` для французької)
2. Відкрийте новий файл у Qt Linguist та заповніть переклади
3. Скомпілюйте його: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Надішліть pull request із файлами `.ts` та `.qm`

Повні інструкції з прикладами містяться у [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Поточні мови:** Англійська плюс німецька, іспанська, французька, хінді, індонезійська, італійська, японська, корейська, польська, португальська (Бразилія), російська, турецька, українська, в'єтнамська, китайська (спрощена) та китайська (традиційна, Тайвань). Оберіть свою у Edit > Preferences. Хочете побачити тут свою мову або покращити наявний переклад? Pull request'и вітаються.
