> **최신 릴리스: [v2.1.4](https://github.com/edenaion/EZ-CorridorKey/releases/tag/v2.1.4):** Windows 핫픽스. 이전 버전이 설치한 결함 있는 FFmpeg 빌드로 인해 가져오기 후 AI 알파 생성이 "failed to read frame"으로 실패하는 문제를 수정합니다([#184](https://github.com/edenaion/EZ-CorridorKey/issues/184)). 이제 가져오기는 자동 재시도와 함께 무결성 검사가 이루어지며, 기본적으로 꺼진 선택적 충돌 보고 기능이 추가되었습니다. [전체 변경 로그](CHANGELOG.md)를 확인하세요.

[Niko Pueringer의 CorridorKey](https://github.com/nikopueringer/CorridorKey)를 위한 전체 데스크톱 GUI입니다. Corridor Digital의 AI 크로마 키어로, 전경을 배경에서 물리적으로 분리해 머리카락, 모션 블러, 반투명 가장자리를 보존합니다.

이 GUI는 CLI 드래그 앤 드롭 워크플로를 완전한 데스크톱 애플리케이션으로 대체하면서도 100% 하위 호환성을 유지합니다. `python main.py --cli`는 여전히 원래 마법사를 실행합니다.

### 데스크톱 앱 설치 프로그램 (권장)

**Python, git, 명령줄을 다루고 싶지 않나요?** 전체 Windows 설치 프로그램, 포터블 exe, macOS `.pkg`를 제공합니다. 완전히 선택 사항이며 무료입니다. 기부는 활발한 개발을 지원합니다.

[![Gumroad에서 다운로드](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

설치 프로그램에는 Python 런타임, AI 모델, GPU 라이브러리 등 필요한 모든 것이 포함됩니다. 별도 설정 없이 설치하고 실행하면 됩니다.

### 터미널 (CLI) 설치 (Windows / macOS / Linux)

1. 이 저장소를 클론하거나 다운로드합니다.
2. 원클릭 경로는 관리형 Python 3.11을 자동으로 준비하고 사용하므로 `1-install`을 쓰기 위해 Python을 미리 설치할 필요가 없습니다.
3. 플랫폼에 맞는 설치 프로그램을 실행합니다:
   ☼ **Windows:** `1-install.bat`를 더블클릭

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. 설치 프로그램은 관리형 Python, 가상 환경, 의존성(사용 가능한 경우 GPU에 맞는 올바른 PyTorch 백엔드 포함), 검증, 모델 다운로드를 모두 처리합니다.
5. 실행하려면 `2-start.bat`(Windows)를 더블클릭하거나 `./2-start.sh`(macOS/Linux)를 실행합니다.

**필수 조건:**

☼ 원클릭 설치 프로그램: 사전 설치된 Python 필요 없음

☼ 수동 설치: [Python 3.10–3.13](https://python.org) (3.14는 아직 지원되지 않음)

☼ **Windows/Linux:** CUDA를 지원하는 NVIDIA GPU (8 GB 이상 VRAM 권장). 드라이버를 최신 상태로 유지하세요. 설치 프로그램은 torch 런타임을 검증하며, 잘못된 백엔드에 조용히 남겨두는 대신 진단 메시지와 함께 중지합니다.

☼ **macOS:** Apple Silicon (M1+). CorridorKey 추론은 MLX를 통해 네이티브로 실행됩니다(MPS보다 1.5–2배 빠름). GPU 집약적인 알파 생성기(SAM2, GVM, VideoMaMa, MatAnyone2)는 MPS에서 실행되지만 훨씬 느립니다. Mac에서는 미리 만든 알파 매트를 가져오는 것을 권장합니다.

**설치 프로그램이 하는 일:**

☼ [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) 확인(OpenEXR에 필요한 C++ 컴파일러). 없으면 자동 설치를 제안

☼ [uv](https://docs.astral.sh/uv/) 설치 및 설치 경로용 관리형 Python 3.11 준비

☼ 프로젝트 폴더에 `.venv` 가상 환경 생성

☼ 플랫폼/GPU에 맞는 올바른 PyTorch 백엔드 설치 및 성공 보고 전에 생성된 torch 런타임 검증

☼ PATH에서 찾지 못하면 [FFmpeg](https://ffmpeg.org/)를 로컬에 다운로드 및 설치(비디오 가져오기에 사용)

☼ CorridorKey 모델 체크포인트 다운로드(383 MB, 필수)

☼ 선택적으로 SAM2 트래킹 지원 설치 및 기본 Base+ 체크포인트(324 MB) 사전 다운로드

☼ 선택적으로 BiRefNet(~940 MB), MatAnyone2(~135 MB), GVM(~6 GB), VideoMaMa(~37 GB) 알파 힌트 생성기 다운로드

☼ 바탕 화면 바로 가기 생성(선택 사항)

**업데이트:**

☼ **Windows 데스크톱 앱 설치 프로그램 사용자:** 앱이 자동으로 업데이트를 확인합니다. 새 버전이 있으면 앱의 업데이트 버튼을 클릭하세요. 가벼운 패치를 다운로드하고 다시 실행합니다.

☼ **macOS 데스크톱 앱 사용자:** 앱이 자동으로 업데이트를 확인합니다. 새 버전이 있으면 앱의 업데이트 버튼을 클릭하세요. 가벼운 패치를 다운로드하고 다시 실행합니다.

☼ **CLI 사용자:** `3-update.bat`(Windows)를 더블클릭하거나 `./3-update.sh`(macOS/Linux)를 실행하세요. git으로 최신 코드를 가져오거나, git을 사용할 수 없으면 ZIP을 다운로드합니다.

> **참고:** GitHub Releases의 업데이트 ZIP(`EZ-CorridorKey-windows-x64.zip`)은 Windows 데스크톱 앱 설치 프로그램 사용자 전용입니다. 기존 설치를 패치합니다. CLI 사용자는 계속 `3-update.bat` / `3-update.sh`를 사용해야 합니다.

### 대체 설치: Docker

Linux 사용자나 원격/클라우드 설정에서는 EZ-CorridorKey를 Docker 안에서 실행하고 noVNC를 통해 브라우저 기반으로 접근할 수 있습니다. 설정 방법은 [docker/README.md](docker/README.md)를 확인하세요. Windows와 macOS에서는 위의 네이티브 설치를 권장합니다.

## 제거

### 데스크톱 앱 설치 프로그램

**Windows:**
☼ 설정 > 앱 > 설치된 앱 열기

☼ **EZ-CorridorKey**를 찾아 제거 클릭

☼ 이렇게 하면 애플리케이션과 시작 메뉴 바로 가기가 제거됩니다

☼ 프로젝트와 다운로드한 모델은 `%APPDATA%\EZ-CorridorKey\`에 저장됩니다. 모든 사용자 데이터를 제거하려면 해당 폴더를 삭제하세요.

**macOS:**
☼ `/Applications/EZ-CorridorKey.app`을 휴지통으로 드래그

☼ 프로젝트, 환경설정, 다운로드한 모델은 `~/Library/Application Support/EZ-CorridorKey/`에 저장됩니다. 모든 사용자 데이터를 제거하려면 해당 폴더를 삭제하세요.

### CLI 설치 (git clone)

☼ 클론한 저장소 폴더를 삭제하세요(예: `EZ-CorridorKey/`). 여기에는 `.venv` 가상 환경, 다운로드한 모델, `Projects/` 안의 모든 프로젝트 데이터가 포함됩니다.

☼ 설치 중 바탕 화면 바로 가기를 만들었다면 수동으로 삭제하세요.

☼ CLI 설치는 시스템 수준 파일을 수정하지 않습니다. 추가로 정리할 것은 없습니다.

### Hugging Face 모델 캐시

일부 선택 모델(BiRefNet, SAM2)은 Hugging Face Hub를 통해 다운로드되어 프로젝트 폴더 밖에 캐시됩니다. 해당 디스크 공간을 회수하려면:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

## 빠른 시작

### 1. 가져오기

환영 화면에 비디오 파일을 드롭하세요(또는 파일 > 클립 가져오기 > 비디오 가져오기). 비디오는 자동으로 고품질 이미지 시퀀스로 추출됩니다:

1. **FFmpeg가** 각 프레임을 **EXR half-float**(ZIP16 압축)로 추출
2. **재압축 패스가** ZIP16 → **DWAB**(VFX 표준 손실 압축, 약 4배 작음)으로 변환

이 2패스 방식은 비디오 디코더의 전체 부동소수점 정밀도를 보존합니다. 8비트 양자화와 밴딩이 없습니다. 8비트 소스 비디오도 이점을 얻습니다. FFmpeg의 내부 YUV→RGB 변환이 float로 유지되어 정수 파이프라인에서 누적되는 반올림 오류를 피하기 때문입니다. DWAB 압축 EXR 프레임은 보통 PNG와 비슷한 크기이며 16비트 half-float 동적 범위를 유지합니다.

재압축은 별도 프로세스에서 실행되므로 추출 중에도 UI가 완전히 반응합니다.

### 2. 알파 힌트 생성

클립은 **RAW** 상태(회색 배지)로 시작합니다. 추론을 실행하기 전에 알파 힌트가 필요합니다.

**옵션 A: 크로마 키 (수동, AI 모델 없음):**
파라미터 패널에서 **CHROMA KEY**를 클릭하세요. 그린 스크린 또는 블루 스크린 푸티지용 색상 차이 키어입니다. **E**를 눌러 스포이드를 활성화한 뒤 화면을 드래그하여 색상 범위를 샘플링합니다. Key Strength, Clip Black, Clip White를 조정해 매트를 다듬습니다. **GENERATE**를 클릭해 알파 힌트를 생성합니다.

**옵션 B: 원클릭 알파 생성기:**
☼ **GVM Auto**: 파라미터 패널에서 **GVM AUTO**를 클릭하세요. 인물이 있는 대부분의 그린 스크린 푸티지에서 잘 작동합니다.

☼ **BiRefNet**(권장): 파라미터 패널에서 **BIREFNET**을 클릭하세요. 빠르고 정확하며 넓은 범위의 푸티지에서 잘 작동합니다.


**옵션 C: MatAnyone2 / VideoMaMa:**
이 모델들은 마스크 힌트가 필요합니다. 제공 방법은 두 가지입니다:

*C1. 페인트 + 트래킹 (처음부터):*
1. **1**을 눌러 전경 모드(초록)를 활성화
2. 몇 개의 키프레임에서 피사체 위에 칠하기
3. **2**를 눌러 배경 모드(빨강)로 전환
4. 배경 영역 위에 칠하기
5. **TRACK MASK**를 클릭해 조밀한 SAM2 마스크 트랙 생성
6. 파라미터 패널에서 **MATANYONE2** 또는 **VIDEOMAMA** 클릭

*C2. 마스크 가져오기 (직접 준비한 파일):*
1. **VIDEOMAMA** 옆의 **+** 버튼을 클릭해 미리 만든 마스크 시퀀스 또는 비디오 가져오기
2. **MATANYONE2** 또는 **VIDEOMAMA**를 클릭해 모델 실행

**옵션 D: 알파 가져오기 (직접 준비한 파일):**
다른 도구(Rotobrush, Silhouette, Resolve, Nuke 등)에서 만든 알파 매트가 이미 있다면 파라미터 패널에서 **IMPORT ALPHA**를 클릭하고 이미지 폴더 또는 매트 비디오 파일을 선택하세요.

☼ 지원 이미지 형식: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ 지원 알파 비디오 경로: 일반 클립 가져오기가 받는 표준 비디오 파일(예: **MOV** 또는 **MP4**)

☼ 이미지와 보이는 매트 비디오는 **grayscale**이어야 합니다(흰색 = 전경, 검정 = 배경)

☼ 프레임 수는 입력 시퀀스와 일치해야 합니다

☼ PNG가 아닌 스틸 이미지는 가져올 때 자동으로 grayscale PNG로 변환됩니다

☼ 가져온 알파 비디오는 grayscale `AlphaHint/*.png` 프레임으로 디코딩되어 이미지 시퀀스 힌트와 같은 후속 경로를 따릅니다

☼ 가져온 파일은 클립의 `AlphaHint/` 폴더로 복사되고 클립은 **READY** 상태로 진행됩니다

언제든지 다시 가져올 수 있습니다. 클립에 이미 알파 힌트가 있으면 덮어쓸지 묻습니다.

### 3. 추론 실행

클립은 이제 **READY**(노란 배지)입니다. 필요에 따라 파라미터를 조정한 뒤 **RUN INFERENCE**(또는 **Ctrl+R**)를 클릭하세요.

### 일괄 파이프라인 (무인)

I/O 트레이에서 여러 클립을 선택(Ctrl+클릭 또는 Shift+클릭)한 뒤 상태 표시줄에서 **RUN PIPELINE**을 클릭하세요. 시스템이 자동으로:

1. 각 클립의 상태와 페인트 스트로크 감지
2. 페인트된 클립에 대해 **TRACK MASK** 실행 후 **VideoMaMa** 실행
3. 페인트되지 않은 RAW 클립에 대해 **GVM Auto** 실행
4. 각 알파 생성이 끝난 뒤 **추론** 연결

전체 파이프라인은 취소 가능(Esc)하고 체크포인트를 남깁니다. 중단된 경우 다시 시작하면 각 클립이 중단된 지점부터 이어집니다.

### 4. 검토

결과를 확인하려면 보기 모드를 전환하세요:

☼ **COMP**: 체커보드 위 키

☼ **FG**: 초록 프린징 확인

☼ **MATTE**: 알파 품질 확인

☼ **PROCESSED**: 제작용 RGBA

출력은 추론 중 프로젝트의 `Output/` 하위 디렉터리에 기록됩니다(설정 가능. [사용자 지정 출력 디렉터리](#사용자-지정-출력-디렉터리) 참조).

## 프레임 스크러버

듀얼 뷰어 아래의 스크러버는 다음을 제공합니다:

☼ **전송 버튼:** 첫 프레임, 한 프레임 뒤로, 재생/일시 정지, 한 프레임 앞으로, 마지막 프레임

☼ **재생은 제한됨:** 스페이스바를 누르면 푸티지가 3FPS의 하드코딩된 속도로 재생됩니다. 파일이 크기 때문에 의도된 동작입니다.

☼ **커버리지 바:** 페인트 스트로크(초록), 알파 힌트(흰색), 추론 출력(노란색)이 있는 프레임을 보여주는 3개의 색상 코딩 레인

## 하드웨어 요구 사항

### Windows / Linux (NVIDIA CUDA)

|          | 최소          | 권장               | 여유 있는 구성                  |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 GB          | 12 GB              | 16 GB+                        |
| **GPU**  | NVIDIA (CUDA) | Ampere+ (RTX 30xx) | Ada/Blackwell (RTX 40xx/50xx) |

#### VRAM 모드

| 모드                  | VRAM 사용량 | 4K 속도    | 작동 방식                              |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | 풀 프레임 리파이너, 전체 torch.compile   |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | 512×512 타일 리파이너, 선택적 컴파일 |

모드는 사용 가능한 VRAM에서 자동 감지됩니다. `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`로 재정의할 수 있습니다.

### macOS (Apple Silicon)

|             | 최소          | 권장                                             |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (corridorkey-mlx가 설치되어 있으면 자동 감지) |

CorridorKey 추론은 사용 가능한 가장 빠른 백엔드를 자동 선택합니다. `corridorkey-mlx`와 `.safetensors` 체크포인트가 있으면 MLX(1.5–2배 빠름), 아니면 PyTorch MPS를 사용합니다. `CORRIDORKEY_BACKEND=torch|mlx|auto`로 재정의할 수 있습니다.

알파 생성기(SAM2, GVM, VideoMaMa, MatAnyone2)는 항상 PyTorch MPS에서 실행됩니다. 이 모델들에는 MLX 포트가 없습니다. Mac에서 가장 좋은 경험을 위해 After Effects, DaVinci Resolve, Nuke에서 만든 알파 매트를 가져오세요.

> v2.0.0 및 소스 설치에 적용됩니다. 2.1.4와 같은 패치 릴리스는 새 Mac 설치 프로그램이 아닌 앱 내 자동 업데이트로 제공됩니다.

## 현지화

EZ-CorridorKey는 모든 언어로 번역할 수 있습니다. 모든 UI 문자열은 표준 Qt `.ts` 파일로 추출되어 번역자가 [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html)(무료) 또는 텍스트 편집기로 수정할 수 있습니다.

**언어를 추가하는 방법:**

1. `ui/translations/corridorkey_en.ts`를 언어 코드로 복사합니다(예: 프랑스어는 `corridorkey_fr.ts`)
2. 새 파일을 Qt Linguist에서 열고 번역을 채웁니다
3. 컴파일합니다: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. `.ts`와 `.qm` 파일을 모두 포함한 pull request를 제출합니다

예제와 전체 지침은 [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md)에 있습니다.

**현재 언어:** 영어 외 독일어, 스페인어, 프랑스어, 힌디어, 인도네시아어, 이탈리아어, 일본어, 한국어, 폴란드어, 포르투갈어(브라질), 러시아어, 터키어, 우크라이나어, 베트남어, 중국어(간체), 중국어(번체, 대만). 편집 > 환경설정에서 언어를 선택하세요. 내 언어를 여기에 보고 싶거나 번역을 개선하고 싶으신가요? PR을 환영합니다.
