## Instalacja

**Wolisz wideo?** Obejrzyj [samouczek krok po kroku](https://www.youtube.com/watch?v=ezg_1oQzz7w) od twórcy aplikacji:

<a href="https://www.youtube.com/watch?v=ezg_1oQzz7w"><img src="https://img.youtube.com/vi/ezg_1oQzz7w/maxresdefault.jpg" alt="EZ-CorridorKey video tutorial" width="640"></a>

### Instalator aplikacji desktopowej (zalecane)

**Nie chcesz zajmować się Pythonem, gitem ani wierszem poleceń?** Dostępny jest pełny instalator Windows, przenośny exe i macOS `.pkg`. To całkowicie opcjonalne i darmowe - darowizny pomagają utrzymać aktywny rozwój.

[![Pobierz na Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

Instalator zawiera wszystko: runtime Pythona, modele AI i biblioteki GPU. Nie wymaga konfiguracji. Po prostu zainstaluj i uruchom.

### Instalacja z terminala (CLI) (Windows / macOS / Linux)

1. Sklonuj albo pobierz to repozytorium.
2. Ścieżka one-click automatycznie przygotowuje i używa zarządzanego Pythona 3.11, więc nie musisz wcześniej instalować Pythona tylko po to, aby użyć `1-install`.
3. Uruchom instalator dla swojej platformy:
   ☼ **Windows:** Kliknij dwukrotnie `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. Instalator obsługuje wszystko: zarządzany Python, środowisko wirtualne, zależności (w tym właściwy backend PyTorch dla Twojego GPU, jeśli jest dostępny), weryfikację i pobieranie modeli.
5. Uruchamianie: kliknij dwukrotnie `2-start.bat` (Windows) albo `./2-start.sh` (macOS/Linux).

**Wymagania wstępne:**

☼ Dla instalatora one-click: wstępnie zainstalowany Python nie jest wymagany

☼ Dla instalacji ręcznych: [Python 3.10–3.13](https://python.org) (3.14 nie jest jeszcze obsługiwany)

☼ **Windows/Linux:** GPU NVIDIA z obsługą CUDA (zalecane 8 GB+ VRAM). Aktualizuj sterownik. Instalator weryfikuje runtime torch i zatrzyma się z diagnostyką, zamiast po cichu zostawić zły backend.

☼ **macOS:** Apple Silicon (M1+). Inferencja CorridorKey działa natywnie przez MLX (1,5-2x szybciej niż MPS). Intensywne GPU generatory alfy (SAM2, GVM, VideoMaMa, MatAnyone2) działają na MPS, ale są znacznie wolniejsze. Na Macu zaleca się import gotowych alpha matte.

**Co robi instalator:**

☼ Sprawdza [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (kompilator C++ wymagany przez OpenEXR) i proponuje autoinstalację, jeśli ich brakuje

☼ Instaluje [uv](https://docs.astral.sh/uv/) i przygotowuje zarządzanego Pythona 3.11 dla ścieżki instalatora

☼ Tworzy środowisko wirtualne `.venv` w folderze projektu

☼ Instaluje właściwy backend PyTorch dla Twojej platformy/GPU i weryfikuje powstały runtime torch przed zgłoszeniem sukcesu

☼ Pobiera i instaluje lokalnie [FFmpeg](https://ffmpeg.org/), jeśli nie znajdzie go na PATH (używany do importu wideo)

☼ Pobiera checkpoint modelu CorridorKey (383 MB, wymagany)

☼ Opcjonalnie instaluje obsługę śledzenia SAM2 i pobiera domyślny checkpoint Base+ (324 MB)

☼ Opcjonalnie pobiera generatory alpha hint BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB) i VideoMaMa (~37 GB)

☼ Tworzy skrót na pulpicie (opcjonalnie)

**Aktualizacja:**

☼ **Użytkownicy instalatora Windows Desktop App:** Aplikacja automatycznie sprawdza aktualizacje. Gdy pojawi się nowa wersja, kliknij przycisk aktualizacji w aplikacji. Pobierze lekki patch i uruchomi się ponownie.

☼ **Użytkownicy macOS Desktop App:** Aplikacja automatycznie sprawdza aktualizacje. Gdy pojawi się nowa wersja, kliknij przycisk aktualizacji w aplikacji. Pobierze lekki patch i uruchomi się ponownie.

☼ **Użytkownicy CLI:** Kliknij dwukrotnie `3-update.bat` (Windows) albo uruchom `./3-update.sh` (macOS/Linux). To pobiera najnowszy kod przez git albo ZIP, jeśli git nie jest dostępny.

> **Uwaga:** ZIP aktualizacji w GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) jest tylko dla użytkowników Windows Desktop App Installer. Łata istniejącą instalację. Użytkownicy CLI powinni nadal używać `3-update.bat` / `3-update.sh`.

### Alternatywna instalacja: Docker

Dla użytkowników Linuksa albo konfiguracji zdalnych/chmurowych EZ-CorridorKey może działać w Dockerze z dostępem przez przeglądarkę dzięki noVNC. Instrukcje konfiguracji są w [docker/README.md](docker/README.md). Natywna instalacja powyżej jest zalecana dla Windows i macOS.

## Odinstalowanie

### Instalator aplikacji desktopowej

**Windows:**
☼ Otwórz Settings > Apps > Installed Apps

☼ Znajdź **EZ-CorridorKey** i kliknij Uninstall

☼ To usuwa aplikację i skrót w menu Start

☼ Projekty i pobrane modele są przechowywane w `%APPDATA%\EZ-CorridorKey\`. Usuń ten folder, aby usunąć wszystkie dane użytkownika.

**macOS:**
☼ Przeciągnij `/Applications/EZ-CorridorKey.app` do Kosza

☼ Projekty, preferencje i pobrane modele są przechowywane w `~/Library/Application Support/EZ-CorridorKey/`. Usuń ten folder, aby usunąć wszystkie dane użytkownika.

### Instalacja CLI (git clone)

☼ Usuń sklonowany folder repozytorium (np. `EZ-CorridorKey/`). Obejmuje to środowisko wirtualne `.venv`, pobrane modele i wszystkie dane projektów w `Projects/`.

☼ Jeśli podczas instalacji utworzono skrót na pulpicie, usuń go ręcznie.

☼ Instalacja CLI nie modyfikuje plików systemowych. Nie ma nic więcej do czyszczenia.

### Cache modeli Hugging Face

Niektóre opcjonalne modele (BiRefNet, SAM2) są pobierane przez Hugging Face Hub i cache'owane poza folderem projektu. Aby odzyskać to miejsce na dysku:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Ten cache jest współdzielony przez wszystkie aplikacje używające Hugging Face. Jeśli używasz innych narzędzi AI, usuń tylko konkretne foldery modeli (np. `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`), a nie cały katalog `hub/`.

## Szybki start

### 1. Import

Upuść plik wideo na ekran powitalny (albo File > Import Clips > Import Video). Wideo zostanie automatycznie wyodrębnione do wysokiej jakości sekwencji obrazów:

1. **FFmpeg wyodrębnia** każdą klatkę jako **EXR half-float** (kompresja ZIP16)
2. **Przebieg rekompresji** konwertuje ZIP16 → **DWAB** (standardowa dla VFX kompresja stratna, ~4× mniejsza)

Ten dwuprzebiegowy proces zachowuje pełną precyzję floating-point z dekodera wideo: bez kwantyzacji 8-bit, bez bandingu. Nawet materiał źródłowy 8-bit zyskuje, ponieważ wewnętrzna konwersja YUV→RGB w FFmpeg pozostaje w float, unikając błędów zaokrągleń narastających w pipeline'ach integer. Klatki EXR skompresowane DWAB mają zwykle rozmiar porównywalny z PNG, a zachowują dynamiczny zakres 16-bit half-float.

Rekompresja działa w osobnym procesie, więc UI pozostaje w pełni responsywne podczas ekstrakcji.

### 2. Wygeneruj wskazówkę alfa

Klip zaczyna w stanie **RAW** (szara etykieta). Przed uruchomieniem inferencji potrzebujesz alpha hint.

**Opcja A - Chroma Key (ręcznie, bez modelu AI):**
Kliknij **CHROMA KEY** w panelu parametrów. To keyer oparty na różnicy kolorów dla materiału green screen albo blue screen. Naciśnij **E**, aby aktywować pipetę, a potem przeciągnij po ekranie, aby pobrać zakres kolorów. Dostosuj Key Strength, Clip Black i Clip White, aby dopracować matte. Kliknij **GENERATE**, aby utworzyć alpha hint.

**Opcja B - generatory alfy jednym kliknięciem:**
☼ **GVM Auto** - kliknij **GVM AUTO** w panelu parametrów. Działa bardzo dobrze dla większości materiałów green screen z ludźmi.

☼ **BiRefNet** (zalecane) - kliknij **BIREFNET** w panelu parametrów. Szybki, dokładny i dobrze działa na szerokim zakresie materiału.


**Opcja C - MatAnyone2 / VideoMaMa:**
Te modele potrzebują mask hint. Są dwa sposoby jego dostarczenia:

*C1. Malowanie + śledzenie (od zera):*
1. Naciśnij **1**, aby aktywować tryb foreground (zielony)
2. Namaluj podmiot na kilku kluczowych klatkach
3. Naciśnij **2**, aby przełączyć na tryb background (czerwony)
4. Namaluj obszary tła
5. Kliknij **ŚLEDŹ MASKĘ**, aby wygenerować gęsty ślad maski SAM2
6. Kliknij **MATANYONE2** albo **VIDEOMAMA** w panelu parametrów

*C2. Import maski (własna):*
1. Kliknij przycisk **+** obok **VIDEOMAMA**, aby zaimportować gotową sekwencję masek albo wideo
2. Kliknij **MATANYONE2** albo **VIDEOMAMA**, aby uruchomić model

**Opcja D - Import Alpha (własna):**
Jeśli masz już alpha matte z innego narzędzia (Rotobrush, Silhouette, Resolve, Nuke itd.), kliknij **IMPORT ALPHA** w panelu parametrów i wybierz folder obrazów albo plik wideo z matte.

☼ Obsługiwane formaty obrazów: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Obsługiwana ścieżka alpha-video: standardowe pliki wideo akceptowane przez zwykły importer klipów (np. **MOV** albo **MP4**)

☼ Obrazy i widoczne filmy matte powinny być **w skali szarości** (biały = foreground, czarny = background)

☼ Liczba klatek powinna odpowiadać sekwencji wejściowej

☼ Obrazy inne niż PNG są automatycznie konwertowane do PNG w skali szarości podczas importu

☼ Zaimportowane filmy alfa są dekodowane do klatek `AlphaHint/*.png` w skali szarości, aby używały tej samej dalszej ścieżki co wskazówki z sekwencji obrazów

☼ Zaimportowane pliki są kopiowane do folderu `AlphaHint/` klipu, a klip przechodzi do stanu **READY**

Możesz importować ponownie w dowolnym momencie. Jeśli klip ma już alpha hints, aplikacja zapyta, czy je nadpisać.

### 3. Uruchom inferencję

Klip jest teraz **READY** (żółta etykieta). Dostosuj parametry według potrzeb, a potem kliknij **RUN INFERENCE** (albo **Ctrl+R**).

### Pipeline wsadowy (bez nadzoru)

Wybierz wiele klipów w tacy I/O (Ctrl+klik albo Shift+klik), a następnie kliknij **RUN PIPELINE** na pasku stanu. System automatycznie:

1. Wykrywa stan każdego klipu i pociągnięcia pędzlem
2. Uruchamia **ŚLEDŹ MASKĘ**, a następnie **VideoMaMa** dla pomalowanych klipów
3. Uruchamia **GVM Auto** dla niepomalowanych klipów RAW
4. Łączy **inferencję** po zakończeniu każdego generowania alfy

Cały pipeline można anulować (Esc) i checkpointować. Po przerwaniu ponowne uruchomienie zaczyna od miejsca, w którym każdy klip skończył.

### 4. Przegląd

Przełączaj tryby widoku, aby sprawdzić wyniki:

☼ **COMP** - key na tle checkerboard

☼ **FG** - sprawdź zielone fringing

☼ **MATTE** - sprawdź jakość alfy

☼ **PROCESSED** - produkcyjne RGBA

Output jest zapisywany w podkatalogach `Output/` projektu podczas inferencji (konfigurowalne, zobacz [Niestandardowy katalog wyjściowy](#niestandardowy-katalog-wyjściowy)).

## Wymagania sprzętowe

### Windows / Linux (NVIDIA CUDA)

|          | Minimum       | Zalecane           | Komfortowo                    |
| -------- | ------------- | ------------------ | ----------------------------- |
| **VRAM** | 8 GB          | 12 GB              | 16 GB+                        |
| **GPU**  | NVIDIA (CUDA) | Ampere+ (RTX 30xx) | Ada/Blackwell (RTX 40xx/50xx) |

#### Tryby VRAM

| Tryb                  | Użycie VRAM | Szybkość 4K | Jak działa                              |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Pełnoklatkowy refiner, pełny torch.compile |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | Refiner kafelkowy 512×512, selektywny compile |

Tryb jest automatycznie wykrywany z dostępnej VRAM. Nadpisz przez `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

> Dotyczy v2.0.0 i instalacji ze źródeł. Wydania łatek, takie jak 2.1.2, są dostarczane jako automatyczna aktualizacja w aplikacji, a nie jako nowy instalator dla Mac.

|             | Minimum       | Zalecane                                         |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (wbudowany, wykrywany automatycznie)         |

Inferencja CorridorKey automatycznie wybiera najszybszy dostępny backend: wbudowany silnik MLX (około 2x szybciej niż MPS, działa w float16), gdy obecne są checkpointy `.mlx.safetensors`, w przeciwnym razie PyTorch MPS. Oba checkpointy MLX (green i blue) można pobrać przez kreator konfiguracji. Nadpisz przez `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Generatory alfy (SAM2, GVM, VideoMaMa, MatAnyone2) zawsze działają na PyTorch MPS. Nie istnieją porty MLX dla tych modeli. Dla najlepszego doświadczenia na Macu importuj gotowe alpha matte z After Effects, DaVinci Resolve albo Nuke.
