## Installazione

### Installer app desktop (consigliato)

**Non vuoi gestire Python, git o la riga di comando?** Sono disponibili un installer Windows completo, un exe portable e un `.pkg` macOS. Sono del tutto opzionali e gratuiti. Le donazioni aiutano a sostenere lo sviluppo attivo.

[![Scarica su Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

L'installer include tutto: runtime Python, modelli AI, librerie GPU. Non serve configurazione. Installa e avvia.

### Installazione da terminale (CLI) (Windows / macOS / Linux)

1. Clona o scarica questo repository.
2. Il percorso one-click prepara e usa automaticamente Python 3.11 gestito, quindi non devi preinstallare Python solo per usare `1-install`.
3. Avvia l'installer per la tua piattaforma:
   ☼ **Windows:** doppio clic su `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. L'installer gestisce tutto: Python gestito, ambiente virtuale, dipendenze (incluso il backend PyTorch corretto per la tua GPU quando disponibile), verifica e download dei modelli.
5. Per avviare: doppio clic su `2-start.bat` (Windows) oppure `./2-start.sh` (macOS/Linux).

**Prerequisiti:**

☼ Per l'installer one-click: Python preinstallato non richiesto

☼ Per installazioni manuali: [Python 3.10-3.13](https://python.org) (3.14 non è ancora supportato)

☼ **Windows/Linux:** GPU NVIDIA con supporto CUDA (consigliati 8 GB+ di VRAM). Mantieni il driver aggiornato. L'installer verifica il runtime torch e si ferma con diagnostica invece di lasciarti in silenzio sul backend sbagliato.

☼ **macOS:** Apple Silicon (M1+). L'inferenza CorridorKey gira nativamente tramite MLX (1,5-2x più veloce di MPS). I generatori alfa intensivi per GPU (SAM2, GVM, VideoMaMa, MatAnyone2) girano su MPS ma sono molto più lenti. Su Mac è consigliata l'importazione di matte alfa già pronti.

**Cosa fa l'installer:**

☼ Controlla [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (compilatore C++ richiesto da OpenEXR) e offre l'installazione automatica se manca

☼ Installa [uv](https://docs.astral.sh/uv/) e prepara Python 3.11 gestito per il percorso dell'installer

☼ Crea un ambiente virtuale `.venv` nella cartella del progetto

☼ Installa il backend PyTorch corretto per la tua piattaforma/GPU e verifica il runtime torch risultante prima di segnalare il successo

☼ Scarica e installa [FFmpeg](https://ffmpeg.org/) localmente se non lo trova nel PATH (usato per importare video)

☼ Scarica il checkpoint del modello CorridorKey (383 MB, richiesto)

☼ Installa opzionalmente il supporto tracking SAM2 e prescarica il checkpoint Base+ predefinito (324 MB)

☼ Scarica opzionalmente i generatori di hint alfa BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB) e VideoMaMa (~37 GB)

☼ Crea un collegamento sul desktop (opzionale)

**Aggiornamento:**

☼ **Utenti Windows Desktop App Installer:** l'app controlla automaticamente gli aggiornamenti. Quando è disponibile una nuova versione, fai clic sul pulsante di aggiornamento nell'app. Scarica una patch leggera e riavvia.

☼ **Utenti macOS Desktop App:** L'app controlla automaticamente gli aggiornamenti. Quando è disponibile una nuova versione, fai clic sul pulsante di aggiornamento nell'app. Scarica una patch leggera e riavvia.

☼ **Utenti CLI:** doppio clic su `3-update.bat` (Windows) oppure esegui `./3-update.sh` (macOS/Linux). Il comando scarica il codice più recente via git o scarica uno ZIP se git non è disponibile.

> **Nota:** Lo ZIP di aggiornamento su GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) è solo per gli utenti Windows Desktop App Installer. Applica una patch a un'installazione esistente. Gli utenti CLI devono continuare a usare `3-update.bat` / `3-update.sh`.

### Installazione alternativa: Docker

Per utenti Linux o setup remoti/cloud, EZ-CorridorKey può girare dentro Docker con accesso via browser tramite noVNC. Vedi [docker/README.md](docker/README.md) per le istruzioni di configurazione. L'installazione nativa sopra è consigliata per Windows e macOS.

## Disinstallazione

### Installer app desktop

**Windows:**
☼ Apri Impostazioni > App > App installate

☼ Trova **EZ-CorridorKey** e fai clic su Disinstalla

☼ Questo rimuove l'applicazione e il collegamento nel menu Start

☼ I tuoi progetti e i modelli scaricati sono archiviati in `%APPDATA%\EZ-CorridorKey\`. Elimina quella cartella per rimuovere tutti i dati utente.

**macOS:**
☼ Trascina `/Applications/EZ-CorridorKey.app` nel Cestino

☼ I tuoi progetti, le preferenze e i modelli scaricati sono archiviati in `~/Library/Application Support/EZ-CorridorKey/`. Elimina quella cartella per rimuovere tutti i dati utente.

### Installazione CLI (git clone)

☼ Elimina la cartella del repository clonato (per esempio `EZ-CorridorKey/`). Questo include l'ambiente virtuale `.venv`, i modelli scaricati e tutti i dati di progetto dentro `Projects/`.

☼ Se hai creato un collegamento sul desktop durante l'installazione, eliminalo manualmente.

☼ L'installazione CLI non modifica file a livello di sistema. Non c'è altro da pulire.

### Cache modelli Hugging Face

Alcuni modelli opzionali (BiRefNet, SAM2) vengono scaricati tramite Hugging Face Hub e memorizzati in cache fuori dalla cartella del progetto. Per recuperare quello spazio disco:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Questa cache è condivisa da tutte le applicazioni che usano Hugging Face. Se usi altri strumenti AI, elimina solo le cartelle dei modelli specifici (per esempio `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`) invece dell'intera directory `hub/`.

## Avvio rapido

### 1. Importazione

Trascina un file video sulla schermata di benvenuto (oppure File > Import Clips > Import Video). Il video viene estratto automaticamente in una sequenza di immagini di alta qualità:

1. **FFmpeg estrae** ogni fotogramma come **EXR half-float** (compressione ZIP16)
2. **Passaggio di ricompressione** converte ZIP16 -> **DWAB** (compressione lossy standard VFX, ~4x più piccola)

Questo approccio a due passaggi preserva tutta la precisione floating-point del decoder video: niente quantizzazione a 8 bit, niente banding. Anche i video sorgente a 8 bit ne beneficiano perché la conversione interna YUV->RGB di FFmpeg resta in float, evitando gli errori di arrotondamento che si accumulano nelle pipeline intere. I fotogrammi EXR compressi DWAB hanno in genere dimensioni paragonabili a PNG, mantenendo però il range dinamico half-float a 16 bit.

La ricompressione gira in un processo separato, quindi l'interfaccia resta completamente reattiva durante l'estrazione.

### 2. Generazione dell'hint alfa

La clip parte in stato **RAW** (badge grigio). Ti serve un hint alfa prima di avviare l'inferenza.

**Opzione A - Chroma Key (manuale, senza modello AI):**
Fai clic su **CHROMA KEY** nel pannello parametri. È un keyer a differenza colore per footage green screen o blue screen. Premi **E** per attivare il contagocce, poi trascina sullo schermo per campionare un intervallo di colori. Regola Key Strength, Clip Black e Clip White per rifinire il matte. Fai clic su **GENERATE** per produrre l'hint alfa.

**Opzione B - Generatori alfa one-click:**
☼ **GVM Auto** - Fai clic su **GVM AUTO** nel pannello parametri. Funziona molto bene per la maggior parte dei green screen con persone.

☼ **BiRefNet** (consigliato) - Fai clic su **BIREFNET** nel pannello parametri. È veloce, accurato e funziona bene su molti tipi di footage.


**Opzione C - MatAnyone2 / VideoMaMa:**
Questi modelli richiedono un hint maschera. Puoi fornirlo in due modi:

*C1. Paint + Track (da zero):*
1. Premi **1** per attivare la modalità primo piano (verde)
2. Dipingi sopra il soggetto su alcuni fotogrammi chiave
3. Premi **2** per passare alla modalità sfondo (rosso)
4. Dipingi sopra le aree di sfondo
5. Fai clic su **TRACK MASK** per generare una traccia maschera SAM2 densa
6. Fai clic su **MATANYONE2** o **VIDEOMAMA** nel pannello parametri

*C2. Importa maschera (fornisci la tua):*
1. Fai clic sul pulsante **+** accanto a **VIDEOMAMA** per importare una sequenza maschera o un video maschera già pronto
2. Fai clic su **MATANYONE2** o **VIDEOMAMA** per avviare il modello

**Opzione D - Importa alfa (fornisci il tuo):**
Se hai già matte alfa da un altro strumento (Rotobrush, Silhouette, Resolve, Nuke, ecc.), fai clic su **IMPORT ALPHA** nel pannello parametri e scegli una cartella immagini o un file video matte.

☼ Formati immagine supportati: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Percorso alpha-video supportato: file video standard accettati dal normale importatore di clip (per esempio **MOV** o **MP4**)

☼ Le immagini e i video matte visibili devono essere in **scala di grigi** (bianco = primo piano, nero = sfondo)

☼ Il numero di fotogrammi deve corrispondere alla sequenza di input

☼ Le immagini fisse non PNG vengono convertite automaticamente in PNG in scala di grigi durante l'importazione

☼ I video alfa importati vengono decodificati in fotogrammi `AlphaHint/*.png` in scala di grigi, così seguono lo stesso percorso downstream degli hint da sequenza immagini

☼ I file importati vengono copiati nella cartella `AlphaHint/` della clip e la clip passa allo stato **READY**

Puoi reimportare in qualsiasi momento. Se la clip ha già hint alfa, ti verrà chiesto se vuoi sovrascriverli.

### 3. Avvio dell'inferenza

La clip ora è **READY** (badge giallo). Regola i parametri se necessario, poi fai clic su **RUN INFERENCE** (oppure **Ctrl+R**).

### Pipeline batch (non presidiata)

Seleziona più clip nel tray I/O (Ctrl+clic o Shift+clic), poi fai clic su **RUN PIPELINE** nella barra di stato. Il sistema:

1. Rileva lo stato e i tratti di paint di ogni clip
2. Esegue **TRACK MASK** e poi **VideoMaMa** per le clip dipinte
3. Esegue **GVM Auto** per le clip RAW non dipinte
4. Collega l'**inferenza** dopo il completamento di ogni generazione alfa

L'intera pipeline può essere annullata (Esc) e usa checkpoint. Se viene interrotta, il riavvio riprende dal punto in cui ogni clip si era fermata.

### 4. Revisione

Passa tra le modalità di visualizzazione per controllare i risultati:

☼ **COMP** - Key su sfondo a scacchiera

☼ **FG** - Controllo del fringing verde

☼ **MATTE** - Controllo qualità dell'alfa

☼ **PROCESSED** - RGBA di produzione

Gli output vengono scritti durante l'inferenza nelle sottocartelle `Output/` del progetto (configurabile, vedi [Cartella di output personalizzata](#cartella-di-output-personalizzata)).

## Scrubber fotogrammi

Lo scrubber sotto il doppio viewer offre:

☼ **Pulsanti di trasporto:** primo fotogramma, passo indietro, play/pausa, passo avanti, ultimo fotogramma

☼ **Playback LIMITATO:** premendo la barra spaziatrice, il footage viene riprodotto a un tasso hardcoded di 3 FPS. È intenzionale: i file sono grandi.

☼ **Barra di copertura:** tre corsie codificate a colori mostrano quali fotogrammi hanno tratti di paint (verde), hint alfa (bianco) e output di inferenza (giallo)

☼ **Marker In/Out:** premi **I** / **O** per impostare un sottointervallo per l'elaborazione. Quando è impostato, il pulsante RUN cambia in "RUN SELECTED" e il playback va in loop dentro l'intervallo.

#### Modalità VRAM

| Modalità              | Uso VRAM   | Velocità 4K | Come funziona                            |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Refiner full-frame, torch.compile completo |
| **Low-VRAM** (<12 GB) | ~2.5 GB    | ~1.6s/frame | Refiner tiled 512x512, compile selettivo |

La modalità viene rilevata automaticamente dalla VRAM disponibile. Sovrascrivi con `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

> Si applica a v2.0.0 e alle installazioni da sorgente. Le release di patch come la 2.1.2 vengono distribuite come aggiornamento automatico in-app, non come nuovo installer Mac.

|             | Minimo        | Consigliato                                      |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (rilevato automaticamente se corridorkey-mlx è installato) |

L'inferenza CorridorKey seleziona automaticamente il backend più veloce disponibile: MLX (1,5-2x più veloce) quando sono presenti `corridorkey-mlx` e un checkpoint `.safetensors`, altrimenti PyTorch MPS. Sovrascrivi con `CORRIDORKEY_BACKEND=torch|mlx|auto`.

I generatori alfa (SAM2, GVM, VideoMaMa, MatAnyone2) girano sempre su PyTorch MPS. Non esistono port MLX per questi modelli. Per la migliore esperienza Mac, importa matte alfa già pronti da After Effects, DaVinci Resolve o Nuke.

### Aggiornamenti firmati

A partire da v1.10.0, ogni release GitHub include un manifest firmato (`manifest.json` + `manifest.json.sig`). Quando l'updater interno scarica una nuova versione, verifica la firma Ed25519 e l'hash SHA-256 prima di applicare l'aggiornamento. Se uno dei controlli fallisce, l'aggiornamento viene respinto e l'utente viene avvisato.

Questo significa che anche se l'account GitHub fosse compromesso, l'attaccante non potrebbe inviare un aggiornamento malevolo agli utenti installati senza possedere anche la chiave di firma offline.

Il codice di verifica è open source: [`backend/update_verify.py`](backend/update_verify.py).

### Checksum

Ogni release include un file `SHA256SUMS.txt` con l'hash SHA-256 di ogni artefatto di release. Usalo per confermare che il download sia autentico e non corrotto. Metti `SHA256SUMS.txt` nella stessa cartella del file scaricato, poi segui i passaggi per il tuo sistema.

**Windows (PowerShell):**

```powershell
# Nella cartella di download (Shift + clic destro > "Apri finestra PowerShell qui")
Get-FileHash .\EZ-CorridorKey-2.1.0-Windows-x64-Setup.exe -Algorithm SHA256
# Apri SHA256SUMS.txt, trova la riga che termina con quel nome file e confronta l'hash.
# Le due stringhe devono corrispondere esattamente. Se differiscono, non eseguire il file, scaricalo di nuovo.
```

**macOS (Terminale):**

```bash
cd ~/Downloads   # dove hai salvato i file
shasum -a 256 -c SHA256SUMS.txt --ignore-missing
# Atteso: EZ-CorridorKey-2.0.0-macOS-arm64.dmg: OK
# Se dice FAILED, il file è corrotto o manomesso, scaricalo di nuovo.
```

**Linux:**

```bash
cd ~/Downloads
sha256sum -c SHA256SUMS.txt --ignore-missing
# Atteso OK accanto a ogni file scaricato.
```

## Localizzazione

EZ-CorridorKey supporta la traduzione in qualsiasi lingua. Tutte le stringhe dell'interfaccia sono estratte in file Qt `.ts` standard che i traduttori possono modificare con [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (gratuito) o con qualsiasi editor di testo.

**Come aggiungere la tua lingua:**

1. Copia `ui/translations/corridorkey_en.ts` nel codice della tua lingua (per esempio `corridorkey_fr.ts` per il francese)
2. Apri il nuovo file in Qt Linguist e compila le traduzioni
3. Compilalo: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Invia una pull request con entrambi i file `.ts` e `.qm`

Le istruzioni complete con esempi sono in [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Lingue attuali:** inglese oltre a tedesco, spagnolo, francese, hindi, indonesiano, italiano, giapponese, coreano, polacco, portoghese (Brasile), russo, turco, ucraino, vietnamita, cinese (semplificato) e cinese (tradizionale, Taiwan). Scegli la tua in Modifica > Preferenze. Vuoi vedere la tua lingua qui o migliorare una traduzione? Le PR sono benvenute.
