### Trình cài đặt ứng dụng desktop (khuyến nghị)

**Không muốn xử lý Python, git hoặc command line?** Có sẵn trình cài đặt Windows đầy đủ, exe portable và `.pkg` macOS. Hoàn toàn tùy chọn và miễn phí, phần đóng góp giúp hỗ trợ phát triển active.

[![Tải xuống trên Gumroad](https://img.shields.io/badge/Download-Gumroad-ff90e8?style=for-the-badge&logo=gumroad&logoColor=white)](https://edenaion.gumroad.com/)

Trình cài đặt bao gồm mọi thứ: runtime Python, mô hình AI, thư viện GPU. Không cần thiết lập. Chỉ cần cài đặt và chạy.

### Cài đặt terminal (CLI) (Windows / macOS / Linux)

1. Clone hoặc tải xuống repository này.
2. Đường dẫn one-click tự động cấp phát và dùng Python 3.11 được quản lý, nên bạn không cần cài Python trước chỉ để dùng `1-install`.
3. Chạy trình cài đặt cho nền tảng của bạn:
   ☼ **Windows:** Nhấp đúp `1-install.bat`

   ☼ **macOS / Linux:** `chmod +x 1-install.sh && ./1-install.sh`

4. Trình cài đặt xử lý mọi thứ: Python được quản lý, virtual environment, dependency (gồm backend PyTorch đúng cho GPU của bạn khi có), xác minh và tải mô hình.
5. Để khởi chạy: nhấp đúp `2-start.bat` (Windows) hoặc `./2-start.sh` (macOS/Linux).

**Điều kiện cần:**

☼ Với trình cài đặt one-click: không cần Python cài sẵn

☼ Với cài đặt thủ công: [Python 3.10–3.13](https://python.org) (3.14 chưa được hỗ trợ)

☼ **Windows/Linux:** GPU NVIDIA có hỗ trợ CUDA (khuyến nghị VRAM 8 GB+). Giữ driver mới. Trình cài đặt xác minh runtime torch và sẽ dừng với chẩn đoán thay vì âm thầm để bạn dùng sai backend.

☼ **macOS:** Apple Silicon (M1+). Suy luận CorridorKey chạy native qua MLX (nhanh hơn MPS 1,5-2x). Các bộ tạo alpha dùng nhiều GPU (SAM2, GVM, VideoMaMa, MatAnyone2) chạy trên MPS nhưng chậm hơn nhiều. Nên nhập matte alpha làm sẵn trên Mac.

**Trình cài đặt làm gì:**

☼ Kiểm tra [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) (compiler C++ mà OpenEXR cần), và đề xuất tự động cài đặt nếu thiếu

☼ Cài [uv](https://docs.astral.sh/uv/) và cấp phát Python 3.11 được quản lý cho đường dẫn trình cài đặt

☼ Tạo virtual environment `.venv` trong thư mục dự án

☼ Cài backend PyTorch đúng cho nền tảng/GPU của bạn và xác minh runtime torch trước khi báo thành công

☼ Tải xuống và cài [FFmpeg](https://ffmpeg.org/) cục bộ nếu không tìm thấy trên PATH (dùng để nhập video)

☼ Tải checkpoint mô hình CorridorKey (383 MB, bắt buộc)

☼ Tùy chọn: cài hỗ trợ tracking SAM2 và tải trước checkpoint Base+ mặc định (324 MB)

☼ Tùy chọn: tải các bộ tạo gợi ý alpha BiRefNet (~940 MB), MatAnyone2 (~135 MB), GVM (~6 GB) và VideoMaMa (~37 GB)

☼ Tạo lối tắt trên desktop (tùy chọn)

**Cập nhật:**

☼ **Người dùng trình cài đặt ứng dụng desktop Windows:** Ứng dụng tự động kiểm tra cập nhật. Khi có phiên bản mới, nhấp nút cập nhật trong ứng dụng. Ứng dụng tải bản vá nhẹ và khởi chạy lại.

☼ **Người dùng ứng dụng desktop macOS:** Ứng dụng tự động kiểm tra cập nhật. Khi có phiên bản mới, nhấp nút cập nhật trong ứng dụng. Ứng dụng tải bản vá nhẹ và khởi chạy lại.

☼ **Người dùng CLI:** Nhấp đúp `3-update.bat` (Windows) hoặc chạy `./3-update.sh` (macOS/Linux). Lệnh này kéo mã mới nhất qua git, hoặc tải ZIP nếu không có git.

> **Ghi chú:** ZIP cập nhật trên GitHub Releases (`EZ-CorridorKey-windows-x64.zip`) chỉ dành cho người dùng trình cài đặt ứng dụng desktop Windows. Nó vá một bản cài đặt hiện có. Người dùng CLI nên tiếp tục dùng `3-update.bat` / `3-update.sh`.

### Cài đặt thay thế: Docker

Với người dùng Linux hoặc thiết lập remote/cloud, EZ-CorridorKey có thể chạy trong Docker với truy cập qua trình duyệt bằng noVNC. Xem [docker/README.md](docker/README.md) để biết hướng dẫn thiết lập. Cài đặt native ở trên được khuyến nghị cho Windows và macOS.

## Gỡ cài đặt

### Trình cài đặt ứng dụng desktop

**Windows:**
☼ Mở Settings > Apps > Installed Apps

☼ Tìm **EZ-CorridorKey** và nhấp Uninstall

☼ Thao tác này gỡ ứng dụng và lối tắt Start Menu

☼ Dự án và mô hình đã tải được lưu trong `%APPDATA%\EZ-CorridorKey\`. Xóa thư mục đó để xóa toàn bộ dữ liệu người dùng.

**macOS:**
☼ Kéo `/Applications/EZ-CorridorKey.app` vào Trash

☼ Dự án, tùy chọn và mô hình đã tải được lưu trong `~/Library/Application Support/EZ-CorridorKey/`. Xóa thư mục đó để xóa toàn bộ dữ liệu người dùng.

### Cài đặt CLI (git clone)

☼ Xóa thư mục repository đã clone (ví dụ `EZ-CorridorKey/`). Thư mục này gồm virtual environment `.venv`, mô hình đã tải và toàn bộ dữ liệu dự án trong `Projects/`.

☼ Nếu đã tạo lối tắt desktop trong lúc cài đặt, hãy xóa thủ công.

☼ Cài đặt CLI không sửa tệp cấp hệ thống. Không còn gì khác cần dọn.

### Cache mô hình Hugging Face

Một số mô hình tùy chọn (BiRefNet, SAM2) được tải qua Hugging Face Hub và cache bên ngoài thư mục dự án. Để lấy lại dung lượng ổ đĩa:

☼ **Windows:** `%USERPROFILE%\.cache\huggingface\hub\`

☼ **macOS / Linux:** `~/.cache/huggingface/hub/`

Cache này được dùng chung bởi mọi ứng dụng dùng Hugging Face. Nếu bạn dùng công cụ AI khác, chỉ xóa các thư mục mô hình cụ thể (ví dụ `models--ZhengPeng7--BiRefNet`, `models--facebook--sam2.1-hiera-base-plus`) thay vì toàn bộ thư mục `hub/`.

## Bắt đầu nhanh

### 1. Nhập

Thả tệp video vào màn hình chào mừng (hoặc File > Nhập Clip > Nhập Video). Video tự động được trích xuất thành chuỗi ảnh chất lượng cao:

1. **FFmpeg extracts** từng khung hình dưới dạng **EXR half-float** (nén ZIP16)
2. **Recompression pass** chuyển ZIP16 → **DWAB** (nén lossy chuẩn VFX, nhỏ hơn ~4x)

Cách hai lượt này giữ nguyên độ chính xác floating-point đầy đủ từ bộ giải mã video: không lượng tử hóa 8-bit, không banding. Ngay cả video nguồn 8-bit cũng có lợi vì chuyển đổi YUV→RGB nội bộ của FFmpeg vẫn ở dạng float, tránh lỗi làm tròn tích lũy trong pipeline integer. Các khung EXR nén DWAB thường có kích thước tương đương PNG trong khi vẫn giữ dynamic range half-float 16-bit.

Quá trình nén lại chạy trong một tiến trình riêng để UI luôn phản hồi đầy đủ trong lúc trích xuất.

### 2. Tạo gợi ý alpha

Clip của bạn bắt đầu ở trạng thái **RAW** (badge xám). Bạn cần gợi ý alpha trước khi chạy suy luận.

**Tùy chọn A - Chroma Key (thủ công, không dùng mô hình AI):**
Nhấp **CHROMA KEY** trong bảng tham số. Đây là keyer dựa trên khác biệt màu cho footage màn xanh lá hoặc xanh dương. Nhấn **E** để bật eyedropper, rồi kéo qua màn hình để lấy mẫu một dải màu. Điều chỉnh Key Strength, Clip Black và Clip White để tinh chỉnh matte. Nhấp **GENERATE** để tạo gợi ý alpha.

**Tùy chọn B - Bộ tạo alpha một nhấp:**
☼ **GVM Auto** - Nhấp **GVM AUTO** trong bảng tham số. Rất tốt cho hầu hết footage màn xanh lá có người.

☼ **BiRefNet** (khuyến nghị) - Nhấp **BIREFNET** trong bảng tham số. Nhanh, chính xác và hoạt động tốt trên nhiều loại footage.


**Tùy chọn C - MatAnyone2 / VideoMaMa:**
Các mô hình này cần gợi ý mask. Có hai cách cung cấp:

*C1. Vẽ + Track (từ đầu):*
1. Nhấn **1** để bật chế độ tiền cảnh (xanh lá)
2. Vẽ lên chủ thể ở vài key frame
3. Nhấn **2** để chuyển sang chế độ nền (đỏ)
4. Vẽ lên các vùng nền
5. Nhấp **TRACK MASK** để tạo track mask SAM2 dày đặc
6. Nhấp **MATANYONE2** hoặc **VIDEOMAMA** trong bảng tham số

*C2. Nhập mask (dùng dữ liệu của bạn):*
1. Nhấp nút **+** cạnh **VIDEOMAMA** để nhập chuỗi mask hoặc video làm sẵn
2. Nhấp **MATANYONE2** hoặc **VIDEOMAMA** để chạy mô hình

**Tùy chọn D - Nhập Alpha (dùng dữ liệu của bạn):**
Nếu bạn đã có matte alpha từ công cụ khác (Rotobrush, Silhouette, Resolve, Nuke, v.v.), nhấp **IMPORT ALPHA** trong bảng tham số và chọn thư mục ảnh hoặc tệp video matte.

☼ Định dạng ảnh được hỗ trợ: **PNG, JPG, JPEG, TIF, TIFF, EXR**

☼ Đường dẫn video alpha được hỗ trợ: các tệp video chuẩn được trình nhập clip thông thường chấp nhận (ví dụ **MOV** hoặc **MP4**)

☼ Ảnh và video matte hiển thị nên là **grayscale** (trắng = tiền cảnh, đen = nền)

☼ Số lượng khung hình nên khớp với chuỗi input của bạn

☼ Ảnh tĩnh không phải PNG tự động được chuyển thành PNG grayscale khi nhập

☼ Video alpha đã nhập được giải mã thành các khung grayscale `AlphaHint/*.png` để đi theo cùng đường downstream như gợi ý chuỗi ảnh

☼ Các tệp đã nhập được sao chép vào thư mục `AlphaHint/` của clip và clip chuyển sang trạng thái **READY**

Bạn có thể nhập lại bất cứ lúc nào. Nếu clip đã có gợi ý alpha, bạn sẽ được hỏi có muốn ghi đè không.

### 3. Chạy suy luận

Clip của bạn giờ là **READY** (badge vàng). Điều chỉnh tham số nếu cần, rồi nhấp **RUN INFERENCE** (hoặc **Ctrl+R**).

### Batch pipeline (không cần giám sát)

Chọn nhiều clip trong khay I/O (Ctrl+click hoặc Shift+click), rồi nhấp **RUN PIPELINE** trên status bar. Hệ thống tự động:

1. Phát hiện trạng thái và nét vẽ của từng clip
2. Chạy **TRACK MASK** rồi **VideoMaMa** cho clip đã vẽ
3. Chạy **GVM Auto** cho clip RAW chưa vẽ
4. Nối tiếp **suy luận** sau khi mỗi bước tạo alpha hoàn tất

Toàn bộ pipeline có thể hủy (Esc) và có checkpoint. Nếu bị gián đoạn, khởi động lại sẽ tiếp tục từ chỗ từng clip đã dừng.

### 4. Xem lại

Chuyển giữa các chế độ xem để kiểm tra kết quả:

☼ **COMP** - Key trên checkerboard

☼ **FG** - Kiểm tra green fringing

☼ **MATTE** - Kiểm tra chất lượng alpha

☼ **PROCESSED** - RGBA sản xuất

Đầu ra được ghi vào các thư mục con `Output/` của dự án trong lúc suy luận (có thể cấu hình, xem [Thư mục đầu ra tùy chỉnh](#thư-mục-đầu-ra-tùy-chỉnh)).

#### Chế độ VRAM

| Chế độ                  | Mức dùng VRAM | Tốc độ 4K    | Cách hoạt động                             |
| --------------------- | ---------- | ----------- | ---------------------------------------- |
| **Speed** (≥12 GB)    | ~8.7 GB    | ~1.5s/frame | Refiner full-frame, torch.compile đầy đủ   |

Chế độ được tự động phát hiện từ VRAM có sẵn. Override bằng `CORRIDORKEY_OPT_MODE=speed|lowvram|auto`.

### macOS (Apple Silicon)

> Áp dụng cho v2.0.0 và cài đặt từ nguồn. Các bản vá như 2.1.4 được phát hành dưới dạng tự động cập nhật trong ứng dụng, không phải trình cài đặt Mac mới.

|             | Tối thiểu       | Khuyến nghị                                      |
| ----------- | ------------- | ------------------------------------------------ |
| **Chip**    | M1 (8 GB)     | M1 Pro+ (16 GB+)                                 |
| **Backend** | MPS (PyTorch) | MLX (tự động phát hiện)                          |

Suy luận CorridorKey tự động chọn backend nhanh nhất có sẵn: engine MLX tích hợp sẵn (nhanh hơn MPS khoảng 2x, chạy ở float16) khi có checkpoint `.mlx.safetensors`, nếu không thì PyTorch MPS. Cả hai checkpoint MLX cho màn xanh lá và xanh dương đều tải qua trình hướng dẫn thiết lập. Override bằng `CORRIDORKEY_BACKEND=torch|mlx|auto`.

Bộ tạo alpha (SAM2, GVM, VideoMaMa, MatAnyone2) luôn chạy trên PyTorch MPS. Không có bản port MLX cho các mô hình này. Để có trải nghiệm Mac tốt nhất, hãy nhập matte alpha làm sẵn từ After Effects, DaVinci Resolve hoặc Nuke.

Mỗi bản phát hành gồm tệp `SHA256SUMS.txt` liệt kê hash SHA-256 của mọi artifact phát hành. Tải nó cùng tệp đã tải xuống, rồi làm theo các bước cho hệ thống của bạn để xác nhận tải xuống hợp lệ và không bị hỏng.

**Windows (PowerShell):**

```powershell
# Trong thư mục tải xuống (Shift + nhấp phải > "Open PowerShell window here")
Get-FileHash .\EZ-CorridorKey-2.1.0-Windows-x64-Setup.exe -Algorithm SHA256
# Mở SHA256SUMS.txt, tìm dòng kết thúc bằng tên tệp đó, so sánh hash.
# Hai chuỗi phải khớp chính xác. Nếu khác nhau, đừng chạy tệp, hãy tải lại.
```

**macOS (Terminal):**

```bash
cd ~/Downloads   # nơi bạn lưu các tệp
shasum -a 256 -c SHA256SUMS.txt --ignore-missing
# Kết quả mong đợi: EZ-CorridorKey-2.0.0-macOS-arm64.dmg: OK
# Nếu báo FAILED, tệp bị hỏng hoặc bị can thiệp, hãy tải lại.
```

**Linux:**

```bash
cd ~/Downloads
sha256sum -c SHA256SUMS.txt --ignore-missing
# Kết quả mong đợi: OK cạnh mỗi tệp đã tải xuống.
```

## Quyền riêng tư

EZ-CorridorKey không gửi bất kỳ dữ liệu đo từ xa nào theo mặc định. Không có phân tích sử dụng, không theo dõi, mãi mãi.

Hai điều có thể gửi dữ liệu, cả hai đều do bạn kiểm soát:

☼ **Báo cáo sự cố**: khi bạn gửi báo cáo từ menu Trợ giúp, bản tóm tắt chẩn đoán bạn thấy trong hộp thoại cũng có thể được gửi trực tiếp đến nhà phát triển. Một hộp kiểm hiển thị kiểm soát điều này cho từng báo cáo.
☼ **Báo cáo crash**: một nút bật tùy chọn trong Tùy chọn > Quyền riêng tư, tắt theo mặc định. Khi bạn bật nó, chi tiết crash được gửi tự động.

Một báo cáo chứa gì: chi tiết crash, thông tin GPU/driver, phiên bản OS và ứng dụng. Không bao giờ chứa gì: phương tiện của bạn, tên tệp, dự án hoặc thông tin cá nhân. Đường dẫn tệp được ẩn danh trước khi gửi.

## Bản địa hóa

EZ-CorridorKey hỗ trợ dịch sang bất kỳ ngôn ngữ nào. Mọi chuỗi UI được trích xuất vào các tệp Qt `.ts` chuẩn mà người dịch có thể chỉnh bằng [Qt Linguist](https://doc.qt.io/qt-6/linguist-translators.html) (miễn phí) hoặc bất kỳ trình soạn thảo văn bản nào.

**Cách thêm ngôn ngữ của bạn:**

1. Sao chép `ui/translations/corridorkey_en.ts` sang mã ngôn ngữ của bạn (ví dụ `corridorkey_fr.ts` cho French)
2. Mở tệp mới trong Qt Linguist và điền bản dịch
3. Compile: `pyside6-lrelease ui/translations/corridorkey_fr.ts`
4. Gửi pull request với cả hai tệp `.ts` và `.qm`

Hướng dẫn đầy đủ kèm ví dụ nằm trong [`ui/translations/TRANSLATING.md`](ui/translations/TRANSLATING.md).

**Ngôn ngữ hiện tại:** English cùng với tiếng Đức, Tây Ban Nha, Pháp, Hindi, Indonesia, Ý, Nhật, Hàn, Ba Lan, Bồ Đào Nha (Brazil), Nga, Thổ Nhĩ Kỳ, Ukraina, Việt, Trung Quốc (Giản thể) và Trung Quốc (Phồn thể, Đài Loan). Chọn ngôn ngữ của bạn trong Edit > Tùy chọn. Muốn thấy ngôn ngữ của bạn ở đây hoặc cải thiện bản dịch? PR được chào đón.
