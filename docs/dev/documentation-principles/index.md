# Documentation Principles

These principles keep docs accurate, minimal, and maintainable.

## Principles

- [Docs describe what exists](principle-1-no-fiction.md) - Document only current main state.
- [One document, one purpose](principle-2-single-responsibility.md) - One audience and one question per file.
- [Scale by folders](principle-3-scale-by-folders.md) - Promote long docs to folders.
- [Link to code properly](principle-4-link-to-code.md) - Use permalinks for non-doc files.
- [Format and consistency](principle-5-format-style.md) - Avoid low-signal formatting.
- [Structure uniformity](principle-6-structure-uniformity.md) - Keep naming and layout consistent.

## Scope

- Applies to all files in `docs/`.
- Rare exceptions must be explicit and explained in the document.
